# Ночные наблюдения — HDC-AM v6.2 (42M params)

## Конфигурация
- hdc_dim=4096, codebook=8192, nav=512x4
- 42M params, batch=24, accum=5, seq=256
- Данные: ~450MB русского текста (wiki + orca + классика + sberquad)
- FixedCharTokenizer vocab=170

## Хронология

### Запуск (0 мин)
- Loss: 5.1 → 2.72 за 300 iter
- Скорость: 768ms/iter (v6.1 24M была 565ms)
- Данные: подтягиваются включая orca4 (172MB)

### План действий
1. Гнать до BPB 2.0 → добавить иерархический HDC
2. До BPB 1.5 → добавить Thought Loops (рекурсивное рассуждение)
3. Фиксировать генерацию на каждом eval

## Наблюдения и инсайты

### Eval 1 (6 мин): BPB 3.66
- На старте 42M чуть медленнее чем 24M (3.66 vs 3.76 у 24M на 4 мин)
- Больше модели нужен дольше warmup

### Eval 2 (13 мин): BPB 3.22
- 42M: 3.22, 24M на 14 мин была 3.15
- 42M пока НЕ обгоняет 24M. Возможно из-за ratio 6x (мало данных для 42M)
- Gap 0.09 — здоровый
- Ждём 30-60 мин — там 42M должна проявить себя

### Eval 6 (39 мин): BPB 2.68 — 42M ОБГОНЯЕТ 24M
- 42M: 2.68, 24M на этом моменте ~2.70
- Большая модель медленно стартовала но теперь впереди
- Генерация: "не устали позволяет", "возможность", "предполагает"
- Gap 0.13 — нормально, данных хватает пока
- Скорость: 771ms/iter стабильно

### Eval ~104 мин: BPB 2.37 — новый рекорд всех версий!
- 42M обгоняет 24M на 0.10 BPB
- Генерация: "согласно заключить", "конкурировать контекст" — грамматика OK
- Gap 0.13-0.19 — переобучение подбирается, нужно больше данных
- Замедление: -0.04/eval. Плато на ~2.3 ожидается
- Для BPB 2.0 нужно: больше данных + LR decay + возможно иерархический HDC

### ФИНАЛ сессии 1: BPB 2.34 за 120 мин (9185 iter)
- Новый абсолютный рекорд всех версий
- Gap 0.13-0.19 — переобучение из-за ratio 4.9x (42M params, ~210M tokens)
- Генерация: правильные слова, грамматика ОК, смысл слабый

### Перезапуск сессии 2
- Чекпоинт загружен (loss 1.52)
- Скачивается orca5 (120K examples, ~350MB) для улучшения ratio
- Цель: BPB 2.0 за эту сессию

### Сессия 2, 58 мин: BPB 2.32 — ПЛАТО
- BPB прыгает 2.32-2.39, не пробивает 2.3
- Gap 0.17-0.23 — переобучение из-за ratio 4.9x
- orca5 (120K examples) ещё скачивается
- Решение: LR 3e-4 → 1e-4 для стабилизации
- Генерация: "не есть только мужчина" — грамматически верно, бессмысленно

### Сессия 3: LR=1e-4, чекпоинт BPB 2.32
- Ждём: orca5 скачается → ratio вырастет → gap уменьшится
- Если BPB не падает с 1e-4 через 30 мин → нужны данные или архитектурные изменения

### Сессия 3, 58 мин: BPB 2.29 — медленный прогресс
- LR=1e-4 помог: gap стабилизировался 0.15-0.20 (было 0.23)
- BPB: 2.32 → 2.29 за час. Плато на ~2.3
- orca5 скрипт умер от SSH timeout, перезапустил через nohup
- Генерация: "осуществляются на грамматический степени", "чтобы провести" — грамматика OK
- ВЫВОД: 42M params + 210M токенов = потолок ~2.3. Нужно больше данных для пробития 2.0

## Ключевые инсайты за ночь
1. HDC-AM v6.1 (24M+триграммы) → BPB 2.47 за 2 часа. Прорыв по скорости
2. HDC-AM v6.2 (42M) → BPB 2.29 за ~4 часа. Обгоняет 24M
3. Потолок определяется ДАННЫМИ, не архитектурой. Ratio 4.9x = переобучение

5. Генерация на BPB 2.3: грамматика правильная, слова верные, смысл слабый
6. Для BPB 2.0: нужно ratio ≥ 10x = 420M+ символов
7. Для BPB 1.5: + иерархический HDC + thought loops
