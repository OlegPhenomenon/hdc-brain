Как нейронная сеть учится

Олег первый раз в жизни решил приготовить утку в духовке. Рецепта нет. Только эксперименты.

Олег и утка

Олег первый раз в жизни решил приготовить утку в духовке. Рецепта нет. Только эксперименты.

  

    CODE
    
  

  Попытка 1: 200°, 2 часа    → сырая внутри, сухая снаружи
Попытка 2: 160°, 3 часа    → получше, но корочка бледная
Попытка 3: 180°, 2.5 часа  → отлично!

Что сделал Олег?

  1. Попробовал
  2. Оценил результат (хорошо или плохо)
  3. Подкрутил параметры (температуру, время)
  4. Повторил

Нейросеть делает ровно то же. Только:

  • параметров не 2 а 299 миллионов
  • "уток" не 3 а миллиарды примеров
  • "попробовать на вкус" = посчитать одно число (loss)

Loss: одно число которое говорит "насколько плохо"

Прежде чем улучшать — нужно измерить. Это как тесты в Rails: если rspec зелёный — хорошо. Красный — плохо. У нейросети вместо rspec — loss ("потеря").

Запусти:

  

    PYTHON
    
  

  import torch
import torch.nn.functional as F

# Модель предсказывает вероятности для 4 слов
# Эти числа до softmax называются "logits" (логиты)
logits = torch.tensor([[2.0, 8.0, 1.0, 0.5]])
#                        ↑    ↑    ↑    ↑
#                      кот  Париж рыба стол

# Правильный ответ: токен №1 (Париж)
target = torch.tensor([1])

# Считаем loss:
loss = F.cross_entropy(logits, target)
print(f"Loss: {loss.item():.4f}")

# Посмотрим вероятности:
probs = F.softmax(logits, dim=-1)
print(f"Вероятности: {[f'{p:.1%}' for p in probs[0].tolist()]}")

Loss маленький — модель уверенно выбрала "Париж" (logit = 8.0 — самый большой).

Теперь поменяй правильный ответ на "кот" (индекс 0) и запусти снова. Loss вырос — модель ошиблась.

Cross-entropy loss — это название формулы. "Cross" = "перекрёстная" (сравниваем два распределения: предсказание модели и правильный ответ). "Entropy" = "энтропия" (мера неопределённости, из теории информации). По сути — "насколько предсказание не совпало с реальностью." Чем точнее — тем меньше loss.

Формула:

  

    CODE
    
  

  Loss = -log(вероятность правильного ответа)

Каждый символ:

  • log — логарифм. Функция которая превращает маленькие вероятности в большие числа
  • - минус — чтобы loss был положительным (log от числа <1 всегда отрицательный)
  • Если вероятность высокая (0.9) → log(0.9) ≈ -0.1 → loss ≈ 0.1 (хорошо!)
  • Если вероятность низкая (0.001) → log(0.001) ≈ -7 → loss ≈ 7 (плохо!)

Градиент: компас который показывает куда крутить ручки

Олег пробовал наугад. Три попытки — ок. Но если параметров 299 миллионов — наугад не выйдет.

Нужен компас: для каждой ручки знать — крутить вправо или влево?

Этот компас называется градиент (gradient). Слово из математики — буквально "наклон", "уклон."

Представь: ты стоишь на холме в тумане и хочешь спуститься вниз. Не видишь ничего. Но можешь пощупать ногой — куда наклон? Делаешь шаг вниз по склону.

  

    CODE
    
  

     ╱╲
  ╱  ╲       ← ты здесь
 ╱    ╲
╱      ╲     → чувствуешь: наклон вправо
        ╲    → делаешь шаг вправо
         ╲
          ●  ← минимум (loss = 0)

Градиент для каждого параметра отвечает на вопрос: "если чуть-чуть увеличить этот параметр — loss вырастет или упадёт?"

  • Градиент положительный → увеличение параметра увеличивает loss → нужно уменьшать
  • Градиент отрицательный → увеличение параметра уменьшает loss → нужно увеличивать

Формула обновления:

  

    CODE
    
  

  параметр_новый = параметр_старый − lr × градиент

lr — это learning rate ("скорость обучения"). Размер шага. Как в Pagination: per_page определяет сколько записей за раз, lr определяет насколько большой шаг.

Это называется gradient descent — "градиентный спуск." Спускаемся по градиенту вниз, к минимуму loss.

Запусти: градиентный спуск своими глазами

  

    PYTHON
    
  

  import torch

# Задача: найти минимум функции f(w) = (w - 3)²
# Минимум при w = 3 (loss = 0)

w = torch.tensor([10.0], requires_grad=True)
# requires_grad=True — говорит PyTorch: "следи за этим числом"

lr = 0.1   # learning rate — размер шага

print("Шаг |    w    |   loss   | визуализация")
print("─" * 50)
for step in range(25):
    loss = (w - 3) ** 2       # считаем loss
    loss.backward()           # PyTorch считает градиент автоматически

    with torch.no_grad():     # "выключаем" отслеживание на время обновления
        w -= lr * w.grad      # шаг вниз по склону
    w.grad.zero_()            # обнуляем (иначе накапливается)

    if step % 4 == 0:
        bar = "█" * max(1, int(loss.item()))
        print(f" {step:2d}  | {w.item():7.3f} | {loss.item():8.4f} | {bar}")

  

    CODE
    
  

  Шаг |    w    |   loss   | визуализация
──────────────────────────────────────────────────
  0  |  10.000 |  49.0000 | █████████████████████████████████████████████████
  4  |   4.310 |   1.7161 | ██
  8  |   3.176 |   0.0311 | █
 12  |   3.010 |   0.0001 | █
 16  |   3.001 |   0.0000 | █
 20  |   3.000 |   0.0000 | █

w стремится к 3. Loss падает к нулю. Обучение работает!

Backpropagation: градиенты для миллионов параметров

У нас 299 миллионов параметров. Как посчитать градиент для каждого?

Считать по одному (менять каждый параметр на чуть-чуть, смотреть как изменился loss) — 299M × 2 вычислений только для одного шага. Слишком медленно.

Решение: backpropagation — "обратное распространение." Сокращённо "backprop."

Идея: если знаешь как loss зависит от выхода слоя — можно посчитать как он зависит от входа. И так пройти назад по всей цепочке.

  

    CODE
    
  

  Прямой проход:
  вход → Слой_1 → Слой_2 → Слой_3 → loss

Обратный проход (backprop):
  loss → Слой_3 → Слой_2 → Слой_1 → все градиенты известны!

Это как дебаг в Rails. Если у тебя ошибка в response, ты идёшь назад по стеку: controller → service → model → database. Backprop делает то же самое — от loss идёт назад по слоям и вычисляет "вклад" каждого параметра в ошибку.

В PyTorch это одна строчка:

  

    PYTHON
    
  

  loss.backward()   # всё! Все 299M градиентов посчитаны

PyTorch записывает все операции во время прямого прохода ("вычислительный граф") и потом проходит по нему назад. Тебе не нужно писать backprop вручную. Никогда.

Полный цикл обучения

Четыре шага — повторяемые тысячи раз:

  

    PYTHON
    
  

  for step in range(10000):
    # 1. Делаем предсказание и считаем loss
    logits = model(x)
    loss = F.cross_entropy(logits, targets)

    # 2. Считаем градиенты (backprop)
    loss.backward()

    # 3. Делаем шаг: подкручиваем параметры
    optimizer.step()

    # 4. Обнуляем градиенты для следующего шага
    optimizer.zero_grad()

Это как CI/CD pipeline:

  

    CODE
    
  

  1. Запусти тесты              → forward pass + loss
2. Посмотри что сломалось     → backward pass (backprop)
3. Исправь код                → optimizer.step()
4. Очисти кеш                 → zero_grad()
5. Повтори                    → следующий шаг

Optimizer (оптимизатор) — это алгоритм который решает КАК именно крутить ручки. Простейший — "шагни в направлении градиента" (gradient descent). Наш — AdamW — умнее: адаптируется к каждому параметру отдельно. Подробно разберём в главе 19.

Запусти: обучаем модель на тексте

Время написать свою первую обучаемую модель. Создай train_lm.py:

  

    PYTHON
    
  

  import torch
import torch.nn as nn
import torch.nn.functional as F

# ── Данные ──
text = open("input.txt").read() if __import__("os").path.exists("input.txt") else \
    "кот сидит на коврике и смотрит в окно. " * 200

chars = sorted(set(text))
c2i = {c: i for i, c in enumerate(chars)}
i2c = {i: c for c, i in c2i.items()}
data = torch.tensor([c2i[c] for c in text])

VOCAB = len(chars)
CTX = 16         # контекст: 16 символов
EMB = 32         # размер вектора
HIDDEN = 128     # скрытый размер

# ── Модель ──
class CharLM(nn.Module):
    def __init__(self):
        super().__init__()
        self.embed = nn.Embedding(VOCAB, EMB)
        self.l1 = nn.Linear(EMB * CTX, HIDDEN)
        self.l2 = nn.Linear(HIDDEN, VOCAB)

    def forward(self, x):
        # x: (batch, CTX) — индексы символов
        emb = self.embed(x)          # (batch, CTX, EMB)
        flat = emb.view(x.size(0), -1)  # (batch, CTX*EMB)
        h = F.gelu(self.l1(flat))    # (batch, HIDDEN)
        return self.l2(h)            # (batch, VOCAB)

model = CharLM()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

# ── Обучение ──
losses = []
for step in range(1000):
    # Случайный кусок текста
    idx = torch.randint(0, len(data) - CTX - 1, (32,))
    x = torch.stack([data[i:i+CTX] for i in idx])
    y = torch.stack([data[i+CTX] for i in idx])

    # Прямой проход
    logits = model(x)
    loss = F.cross_entropy(logits, y)

    # Обратный проход + шаг
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()

    losses.append(loss.item())
    if step % 200 == 0:
        print(f"Шаг {step:4d}: loss = {loss.item():.3f}")

# ── Генерация ──
print("\n--- Генерация ---")
model.eval()
context = list(text[:CTX])
result = list(context)
for _ in range(200):
    x = torch.tensor([[c2i[c] for c in context]])
    logits = model(x)
    probs = F.softmax(logits[0], dim=0)
    next_char = chars[torch.multinomial(probs, 1).item()]
    result.append(next_char)
    context = context[1:] + [next_char]

print("".join(result))

Запусти: python3 train_lm.py

Ты увидишь:

  1. Loss падает с каждым шагом (модель учится)
  2. В конце — сгенерированный текст (сначала мусор, потом всё больше похоже на настоящий)

Что внутри: nn.Embedding

В коде выше есть nn.Embedding(VOCAB, EMB). Что это?

Это таблица: для каждого символа (буквы) хранится вектор.

  

    CODE
    
  

  nn.Embedding(26, 32)  →  таблица 26 строк × 32 колонки

  "а" → [0.23, -0.15, 0.87, ..., 0.41]   (32 числа)
  "б" → [-0.54, 0.33, 0.12, ..., -0.78]
  "в" → [0.67, 0.89, -0.34, ..., 0.12]
  ...

В Rails это как lookup-таблица: CHAR_VECTORS = { "а" => [...], "б" => [...] }. Только значения не заданы вручную — они обучаются вместе с остальной моделью.

В HDC-Brain это называется codebook (кодовая книга). Вместо 32-мерных векторов — 4096-мерные. Вместо 26 символов — 32 768 токенов. Подробно — в главе 11.

Итоги

Обучение нейросети = 4 шага, повторяемые тысячи раз:

  

    CODE
    
  

  1. forward   → модель делает предсказание, считаем loss
2. backward  → PyTorch считает все градиенты (backprop)
3. step      → оптимизатор подкручивает параметры
4. zero_grad → обнуляем для следующего шага

  

    CODE
    
  

  Loss            = "насколько плохо" (чем меньше — тем лучше)
Градиент        = "куда крутить каждую ручку" (компас)
Backpropagation = "считаем все градиенты за один проход назад"
Learning rate   = "насколько сильно крутить" (размер шага)
Optimizer       = "алгоритм кручения" (Adam, AdamW — подробно в гл. 19)

Ты написал и обучил свою первую нейросеть на тексте. Она генерирует похожий на настоящий текст — из случайного мусора, через тысячу шагов обучения.
