Математика: линейная алгебра и теория информации

Вектор — это просто упорядоченный список чисел:

Вся математика книги в одном месте. Без доказательств — только интуиция, примеры и формулы которые нужны чтобы понять код.

Если ты видишь формулу в коде и не понимаешь что она делает — загляни сюда.

B.1 Векторы — списки чисел

Вектор — это просто упорядоченный список чисел:

  

    CODE
    
  

  a = [3, -1, 2, 5, -4]  ← вектор из 5 чисел
b = [+1, -1, +1, -1, +1]  ← биполярный вектор из 5 чисел

В нашей архитектуре каждый токен — это вектор из 4096 биполярных чисел {-1, +1}.

Длина вектора (норма) — расстояние от нуля:

  

    CODE
    
  

  ‖a‖ = √(a₁² + a₂² + ... + aₙ²)

Пример: ‖[3, 4]‖ = √(9 + 16) = √25 = 5

В Python: torch.norm(a) или a.norm().

Скалярное произведение — "насколько векторы похожи":

  

    CODE
    
  

  a · b = a₁×b₁ + a₂×b₂ + ... + aₙ×bₙ

Пример: [1, 2, 3] · [4, 5, 6] = 1×4 + 2×5 + 3×6 = 4 + 10 + 18 = 32

В Python: (a * b).sum() или torch.dot(a, b).

Косинусное сходство — нормализованная похожесть (от -1 до +1):

  

    CODE
    
  

  cos(θ) = (a · b) / (‖a‖ × ‖b‖)
  • cos = 1.0 → векторы смотрят в одну сторону (одинаковые)
  • cos = 0.0 → векторы перпендикулярны (не похожи)
  • cos = -1.0 → векторы смотрят в противоположные стороны

В Python: torch.nn.functional.cosine_similarity(a, b, dim=0).

B.2 Биполярные векторы и квазиортогональность

Наши токен-векторы биполярные: каждое из 4096 чисел либо -1 либо +1.

Важный факт: два случайных биполярных вектора почти перпендикулярны.

Почему? Каждое слагаемое aᵢ × bᵢ случайно равно +1 или -1 с равной вероятностью. Сумма 4096 таких случайных ±1 стремится к нулю по закону больших чисел.

Математически:

  

    CODE
    
  

  E[a · b] = 0          ← в среднем ноль
Std[a · b] ≈ √4096 = 64   ← разброс

Нормированное сходство:
  cos(θ) = (a · b) / 4096
  Std[cos] = 64/4096 ≈ 0.016

Вывод: с вероятностью >99% cos(θ) будет в диапазоне [-0.05, +0.05]

Это называется квазиортогональность: векторы не строго перпендикулярны, но очень близко.

Практическое следствие: в 4096-мерном пространстве можно хранить 32768 токенов и они почти не будут "мешать" друг другу.

B.3 Матрицы — таблицы чисел

Матрица — прямоугольная таблица чисел:

  

    CODE
    
  

  W = [[1, 2, 3],    ← строка 0
     [4, 5, 6],    ← строка 1
     [7, 8, 9]]    ← строка 2

Размер: 3 × 3 (строки × столбцы)

Кодбук — это матрица: 32768 × 4096 (токенов × измерений).

Умножение матрицы на вектор (самая частая операция в нейросетях):

  

    CODE
    
  

  W × x = y

Если W размера (m × n) и x размера (n,):
  y размера (m,)
  y[i] = строка_i(W) · x = Σⱼ W[i,j] × x[j]

  

    PYTHON
    
  

  # PyTorch:
W = torch.randn(10, 5)  # матрица 10×5
x = torch.randn(5)      # вектор размера 5
y = W @ x               # вектор размера 10

Умножение матрица × матрица:

  

    CODE
    
  

  C = A × B

Если A размера (m × k) и B размера (k × n):
  C размера (m × n)
  C[i, j] = строка_i(A) · столбец_j(B)

  

    PYTHON
    
  

  A = torch.randn(4, 8)   # 4×8
B = torch.randn(8, 6)   # 8×6
C = A @ B               # 4×6

Транспонирование — поворот матрицы: строки становятся столбцами:

  

    CODE
    
  

  A.T[i, j] = A[j, i]

  

    PYTHON
    
  

  A.T  # или A.transpose(-1, -2)

B.4 Функции активации — нелинейности

Без нелинейностей нейросеть — просто линейная комбинация. Нелинейности дают мощь.

GELU — функция которую используем в ControllerBlock:

  

    CODE
    
  

  GELU(x) = x × Φ(x)

где Φ(x) = P(Z ≤ x) для стандартного нормального Z

Проще говоря:

  • При большом положительном x: GELU(x) ≈ x (пропускаем как есть)
  • При большом отрицательном x: GELU(x) ≈ 0 (блокируем)
  • В районе нуля: плавный переход
  

    PYTHON
    
  

  torch.nn.functional.gelu(x)

Sigmoid — превращает любое число в (0, 1):

  

    CODE
    
  

  σ(x) = 1 / (1 + e⁻ˣ)

σ(0)  = 0.5
σ(5)  ≈ 0.993
σ(-5) ≈ 0.007

Используем для gate значений: gate = σ(learned_param).

  

    PYTHON
    
  

  torch.sigmoid(x)

Softmax — превращает вектор в вероятности (сумма = 1):

  

    CODE
    
  

  softmax([1, 2, 3]) = [e¹, e², e³] / (e¹ + e² + e³)
                   ≈ [0.09, 0.24, 0.67]

Используем для финального предсказания следующего токена.

  

    PYTHON
    
  

  torch.softmax(logits, dim=-1)

sign — знак числа:

  

    CODE
    
  

  sign(5.3) = +1
sign(-2.1) = -1
sign(0) = 0

Используем для бинаризации: sign(weight) превращает вещественные веса в {-1, +1}.

Проблема: d/dx sign(x) = 0 везде → нельзя учить обычным градиентным спуском. Решение: STE (см. раздел B.8).

B.5 Нормализация

LayerNorm — нормализует вектор:

  

    CODE
    
  

  Дано: x = [10, 15, 8, 22, 5]   (вектор из 5 чисел)

μ = mean(x) = (10+15+8+22+5)/5 = 12
σ = std(x)  = √(дисперсия) ≈ 5.97

LN(x) = (x - 12) / 5.97 ≈ [-0.33, 0.50, -0.67, 1.68, -1.17]

После нормализации: среднее = 0, стандартное отклонение = 1.

Потом масштабирование: LN(x) = (x - μ)/(σ + ε) × γ + β

Где γ и β — обучаемые параметры (можно восстановить нужный масштаб).

Зачем? Без нормализации числа в нейросети могут "взорваться" (стать огромными) или "схлопнуться" (стать нулями). LayerNorm держит их в разумном диапазоне.

  

    PYTHON
    
  

  nn.LayerNorm(4096)  # нормализует по последнему измерению

B.6 Производные и градиенты

Производная — скорость изменения функции:

  

    CODE
    
  

  f'(x) = "насколько быстро f(x) меняется при изменении x"

Если f(x) = x², то f'(x) = 2x
Если f(2) = 4, то f'(2) = 4: при изменении x на 0.01, f меняется на ≈0.04

Градиент — производная для функции многих переменных:

  

    CODE
    
  

  ∇f(x₁, x₂, x₃) = [∂f/∂x₁, ∂f/∂x₂, ∂f/∂x₃]

Это вектор: каждый элемент показывает как меняется f при изменении соответствующей переменной.

Цепное правило — ключ к backpropagation:

  

    CODE
    
  

  Если y = f(g(x)), то:
  dy/dx = (df/dg) × (dg/dx)

Нейросеть — цепочка функций. Производная раскрывается по цепному правилу через все слои.

Пример:

  

    CODE
    
  

  x → g(x) = x² → h(g) = g + 1 → f(h) = h³

df/dx = (df/dh) × (dh/dg) × (dg/dx)
      = 3h²    ×  1       ×  2x
      = 3(x²+1)² × 2x

B.7 Entropy и BPB

Энтропия — мера "неопределённости":

  

    CODE
    
  

  H(P) = -Σᵢ P(i) × log₂(P(i))

Единица: биты. Смысл: "сколько бит нужно чтобы описать результат".

Примеры:

  

    CODE
    
  

  Монета: P(орёл) = P(решка) = 0.5
H = -(0.5 × log₂(0.5) + 0.5 × log₂(0.5)) = 1 бит  ← нужен 1 бит

Нечестный кубик: P(1) = 0.99, P(2)=...=P(6)=0.002
H ≈ 0.09 бит  ← почти нет неопределённости

32768 равновероятных токенов:
H = log₂(32768) = 15 бит  ← максимальная неопределённость

Cross-Entropy Loss — наш loss:

  

    CODE
    
  

  loss = -log P(правильный_токен)
  • Предсказали правильный токен с вероятностью 0.5 → loss = -log(0.5) = 0.69
  • С вероятностью 0.1 → loss = -log(0.1) = 2.30
  • С вероятностью 0.01 → loss = -log(0.01) = 4.61

BPB (Bits Per Byte):

  

    CODE
    
  

  BPB = loss / log(2)

Переводит loss из натуральных логарифмов в биты. Интерпретация: если бы модель была сжималкой — сколько бит на байт текста.

Ориентиры:

  

    CODE
    
  

  BPB = 1.0 — идеальное предсказание (недостижимо)
BPB = 2.0 — GPT-4 на английском
BPB = 4.0 — хорошая маленькая модель
BPB = 7.5 — почти случайное угадывание
BPB = 15  — полностью случайное (log₂(32768) = 15)

B.8 STE: математика трюка

Проблема: функция sign(x) имеет нулевой градиент везде. Нельзя обучать.

  

    CODE
    
  

  d/dx sign(x) = 0  при x ≠ 0

Если в цепи backpropagation встречается sign() → через неё передаётся градиент × 0 = 0 → все предшествующие параметры не обучаются.

Решение: Straight-Through Estimator (Bengio et al., 2013):

Притворяемся что sign(x) = x (identity function). Тогда производная = 1.

Реализация в PyTorch:

  

    PYTHON
    
  

  hard = weight.sign()   # {-1, +1}

# Трюк:
ste = (hard - weight).detach() + weight

# Разберём:
# .detach() → убирает из графа вычислений
# При backward:
#   d(ste)/d(weight) = d((hard - weight).detach() + weight)/d(weight)
#                    = 0 + 1 = 1
# Градиент течёт через weight как через identity!

# При forward:
# ste = (hard - weight) + weight = hard  ← реально {-1, +1}

STE — приближение. Это не точно математически. Но работает на практике: модель обучается, кодбук бинаризуется.

Почему работает? После обучения большинство весов далеко от нуля (|w| >> 0). Тогда sign(w) ≈ w / |w| — почти линейная функция для больших |w|. STE-приближение становится точнее.

B.9 Attention: математика

Стандартный Scaled Dot-Product Attention:

  

    CODE
    
  

  Attention(Q, K, V) = softmax(Q × Kᵀ / √dₖ) × V

Размерности:

  

    CODE
    
  

  Q: (T, dₖ)   — T запросов (queries)
K: (T, dₖ)   — T ключей (keys)
V: (T, dᵥ)   — T значений (values)

Q × Kᵀ: (T, T)   — матрица попарных сходств
/ √dₖ:           — масштабирование
softmax(·):       — веса (каждая строка суммируется в 1)
× V: (T, dᵥ)     — взвешенная сумма значений

Почему делим на √dₖ?

Скалярное произведение q · k двух dₖ-мерных случайных векторов имеет стандартное отклонение √dₖ. При большом dₖ (например, 4096) значения огромные → softmax коллапсирует на один токен.

  

    CODE
    
  

  dₖ = 4096, без масштабирования: std(q·k) ≈ 64
softmax([64, 1, -3, 8]) ≈ [1.0, 0.0, 0.0, 0.0]  ← победитель берёт всё

dₖ = 4096, с масштабированием / √4096 = /64: std ≈ 1
softmax([1.0, 0.02, -0.05, 0.13]) ≈ [0.43, 0.25, 0.23, 0.29]  ← мягче

Наш HDC Binding Attention заменяет Q × Kᵀ на поэлементное умножение q * k:

  

    CODE
    
  

  binding = q * k   ← форма (B, H, T, head_dim) — НЕТ матрицы T×T
scores = binding.sum(-1)  ← (B, H, T) — не (B, H, T, T)

Это принципиально другая математика: не попарные сходства, а HDC Bind.

B.10 Cosine Learning Rate Schedule

Формула косинусного затухания:

  

    CODE
    
  

  lr(t) = lr_min + (lr_max - lr_min) × (1 + cos(π × progress)) / 2

progress = (t - warmup) / (total - warmup)  ← от 0 до 1

При progress=0: lr = lr_max. При progress=1: lr = lr_min.

Почему косинус, а не просто линейное убывание?

  

    CODE
    
  

  progress →  0.0   0.25  0.5   0.75  1.0
cos(π×p)  →  1.0   0.71  0.0  -0.71 -1.0
lr factor →  1.0   0.85  0.5   0.15  0.0

В начале (0→0.25): lr уменьшается медленно → "шагаем широко пока ищем направление" В середине (0.25→0.75): быстрее убывает → "проходим через сложные места" В конце (0.75→1.0): снова медленно → "тонко подстраиваемся у минимума"

B.11 Weight Decay

Проблема: без ограничений, веса могут расти бесконечно → переобучение.

L2-регуляризация — добавляем штраф к loss:

  

    CODE
    
  

  L_total = L_cross_entropy + (λ/2) × Σ wᵢ²

При оптимизации: ∂L_total/∂wᵢ = ∂L/∂wᵢ + λ × wᵢ

Обновление: wᵢ ← wᵢ × (1 - lr × λ) - lr × ∂L/∂wᵢ

Множитель (1 - lr × λ) < 1 → веса постепенно "усыхают" к нулю.

Проблема Adam + L2: в Adam градиент делится на √v̂ (нормализуется). Если прибавить λ × wᵢ до деления — штраф тоже нормализуется, что неправильно. Для параметров с большой историей градиентов √v̂ большой → эффективный weight decay маленький.

AdamW (правильный): применяет weight decay после адаптации:

  

    CODE
    
  

  AdamW:  wᵢ ← wᵢ × (1 - lr × λ)  - lr × AdamStep(∂L/∂wᵢ)
                 ↑ decay             ↑ adam
            (отдельно!)          (отдельно!)

Weight decay всегда работает с полной силой λ, независимо от истории градиентов.

B.12 Параллельный скан в HDCMemory

Задача: для каждой позиции t вычислить взвешенную накопленную сумму прошлого:

  

    CODE
    
  

  context[t] = Σᵢ₌₀ᵗ decay(i→t) × mass[i] × token[i]

где decay(i→t) = decay[i+1] × decay[i+2] × ... × decay[t]
              = Π_{k=i+1}^{t} decay[k]

Произведение decay от i до t — это нарастающее произведение.

Через логарифмы:

  

    CODE
    
  

  log(decay(i→t)) = log(decay[i+1]) + log(decay[i+2]) + ... + log(decay[t])
                = cumlog[t] - cumlog[i]

где cumlog[t] = Σₖ₌₀ᵗ log(decay[k])  ← накопленная сумма логарифмов

Реализация:

  

    PYTHON
    
  

  log_decay = torch.log(decay.clamp(min=1e-6))  # (B, T, 1)
cumlog    = torch.cumsum(log_decay, dim=1)      # (B, T, 1)

# diff[b, t, i] = cumlog[t] - cumlog[i] = log(decay(i→t))
diff = cumlog.unsqueeze(2) - cumlog.unsqueeze(1)  # (B, T, T)
diff = diff.clamp(max=0)   # затухание только уменьшает, не увеличивает

# Применяем causal mask:
causal = torch.tril(torch.ones(T, T))
diff[causal == 0] = float('-inf')

# Веса:
weights = torch.exp(diff)  # (B, T, T) — decay-веса для каждой пары

# Итоговый контекст:
context = weights @ (mass * tokens)  # (B, T, D)

torch.cumsum — высокоэффективная операция на GPU, O(T) вычислений. Итоговый weights @ tokens — матричное умножение, GEMM на CUDA.
