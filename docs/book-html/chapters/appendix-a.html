Глоссарий всех терминов

Adam — умный оптимизатор. Обычный gradient descent двигает все параметры на одинаковый шаг. Adam запоминает историю для каждого параметра отдельно: если параметр часто получает большие градиенты —

Все термины которые встречаются в книге — в одном месте.

Написан так: сначала простое объяснение человеческими словами, потом точное определение.

А

Adam — умный оптимизатор. Обычный gradient descent двигает все параметры на одинаковый шаг. Adam запоминает историю для каждого параметра отдельно: если параметр часто получает большие градиенты — шаг меньше, если маленькие — шаг больше. Это "адаптивный" в названии. Полное название: Adaptive Moment Estimation. Kingma & Ba (2014).

AdamW — Adam с правильным weight decay. Обычный Adam смешивает weight decay с адаптацией (что неправильно математически). AdamW применяет weight decay отдельно: сначала уменьшает веса, потом делает шаг по градиенту. Буква "W" = Weight decay (correct). Loshchilov & Hutter (2017).

Attention — механизм "внимания". Когда обрабатываем слово, смотрим на все предыдущие слова и решаем: это важно для понимания? Технически: вычисляем схожесть запроса (Q) каждого токена с ключами (K) других токенов, используем схожесть как вес при суммировании значений (V).

Autoregressive — генерация по одному. Модель предсказывает следующий токен, добавляет его к контексту, снова предсказывает... и так до конца. Как рассказывать историю: не знаешь следующее слово пока не придумал предыдущее.

Б

Backpropagation (обратное распространение ошибки) — способ считать градиенты. Посчитали loss → откуда он пришёл? Прошли назад через все операции по цепному правилу: если z = f(y), y = g(x), то dL/dx = dL/dz × dz/dy × dy/dx. Так для каждого параметра получаем: "насколько он виновен в ошибке".

Batch / Батч — группа примеров которые обрабатываются вместе. Вместо одного предложения — сразу 16 предложений. GPU отлично умеет обрабатывать много примеров параллельно. Большой batch → стабильнее градиенты (меньше шума) → лучше обучение.

bf16 / bfloat16 — 16-битный формат чисел. Обычный float32 — 32 бита. Наш bf16 — 16 бит: быстрее, меньше памяти. Секрет: bf16 сохраняет диапазон как у float32 (не переполняется), но жертвует точностью (2 значащих цифры вместо 7). Для нейросетей диапазон важнее точности.

Bind — операция HDC. Создать вектор "А и Б вместе": поэлементно умножить A на B. A * B = C. Результат C квазиортогонален обоим A и B — это и есть "связывание" информации.

Bipolar — пространство {-1, +1}. Каждое из 4096 измерений вектора равно либо -1 либо +1. Отличие от бинарного {0, 1}: можно делать вычитание, есть симметрия, легче считать нормы.

BPB (Bits Per Byte) — метрика качества. Если бы модель была сжималкой — сколько бит тратила бы на каждый байт текста? BPB = loss / log(2). Ориентиры: 1.0 = идеально (недостижимо), 2.0 = GPT-4 на английском, 4.0 = хорошо, 7.5 = случайное угадывание.

BPE (Byte Pair Encoding) — алгоритм создания словаря токенов. Начинаем с символов. Находим самую частую пару символов → объединяем в один токен. Повторяем. Через 32000 итераций — словарь 32K токенов где частые слова = один токен, редкие слова = несколько.

Bundle — операция HDC. "Усреднить" несколько векторов: sign(A + B + C). Результат близок ко всем трём, но не равен ни одному. Как компромисс: "немного от каждого".

В

Validation Loss — потери на тест-данных. Обучаем на одних данных, проверяем на других (которых модель не видела). Если train loss низкий, val loss высокий — модель "зазубрила" тренировку. Validation loss — честная мера качества.

Vocabulary / Словарь — все токены которые знает модель. В v14.1: 32768 токенов. Включает: слова, части слов, знаки препинания, пробелы.

Г

Gate — "вентиль". Число от 0 до 1. Контролирует смешивание: output = (1-gate) × старый + gate × новый. gate=0 → берём старое, gate=1 → берём новое, gate=0.5 → поровну. В ThoughtLoop gate обучается: модель сама решает насколько слушать каждую "мысль".

GELU (Gaussian Error Linear Unit) — нелинейная функция. Похожа на ReLU (max(0, x)) но плавнее: для больших отрицательных x → 0, для больших положительных x → x, в районе нуля — плавный переход. Формула: x × Φ(x) где Φ — CDF нормального распределения.

Gradient / Градиент — вектор который показывает "в каком направлении loss растёт". Если идти против градиента — loss уменьшается. Для каждого параметра: насколько он влияет на loss.

Gradient Accumulation — накопление градиентов. Хотим большой batch, но не влезает в память. Решение: делаем 8 маленьких шагов, накапливаем градиенты (они суммируются), потом один шаг оптимизатора. Математически эквивалентно одному большому батчу.

Gradient Checkpointing — экономия памяти. При forward pass PyTorch хранит все промежуточные результаты (нужны для backward). Много памяти! Checkpointing: не храним → при backward пересчитываем заново. В 2 раза медленнее, в 8 раз меньше памяти. Стоит того.

Gradient Clipping — страховка от "взрыва". Иногда градиенты становятся огромными → веса прыгают → обучение рушится. Решение: если норма градиента > 1.0, масштабируем все градиенты вниз. Направление сохраняется, размер ограничивается.

Д

Decay / Затухание — в HDCMemory: скорость с которой токен "тускнеет" в памяти. Число от 0 до 1. decay=0.9 → токен долго остаётся в памяти. decay=0.1 → быстро забывается. Обучается автоматически.

З

Zero-shot — решение задачи без специальной подготовки. Модель не видела примеров этой задачи, но справляется благодаря общему пониманию языка. Например: GPT-4 переводит текст хотя специально переводу не обучалась.

И

Inference — использование обученной модели. В отличие от обучения: нет backward pass, нет обновления весов. Просто: токены → модель → предсказание. Обычно быстрее обучения.

К

Causal Mask — маска "не заглядывать в будущее". Для языковой модели токен на позиции 5 не должен видеть токены 6, 7, 8... Реализуется нижнетреугольной матрицей: mask[i,j] = 1 если j ≤ i.

Кодбук (Codebook) — таблица векторов для токенов. vocab_size × dim матрица. Строка i = вектор токена i. В v14.1: 32768 × 4096 биполярных чисел = 131M параметров.

Cosine Schedule — расписание learning rate. Плавно уменьшается по косинусу: начинаем с maxlr, заканчиваем minlr. Форма: min_lr + (max_lr - min_lr) × 0.5 × (1 + cos(π × progress)). С warmup в начале.

Cross-Entropy Loss — функция потерь. Что модель предсказала вероятность правильного ответа? -log(p_правильного). Если предсказали правильный токен с вероятностью 1.0 → loss=0. С вероятностью 0.5 → loss=0.69. С вероятностью 0.01 → loss=4.6. Меньше = лучше.

KV Cache — кэш для ускорения inference. В стандартном трансформере при добавлении нового токена незачем пересчитывать K и V для старых токенов. Кэшируем → намного быстрее. В нашей архитектуре не реализован (из-за HDCMemory).

Л

LayerNorm / Нормализация слоя — нормализация внутри одного примера. Берём вектор из 4096 чисел, вычитаем среднее, делим на стандартное отклонение → числа в разумном диапазоне. Потом масштабируем и сдвигаем (обучаемые γ и β). Нужно для стабильного обучения.

Learning Rate / Скорость обучения — насколько большой шаг делаем при обновлении параметров. w -= lr × gradient. Слишком большой → прыгаем через минимум → нестабильность. Слишком маленький → учимся вечность.

LLM (Large Language Model) — большая языковая модель. "Большая" = много параметров (от миллиардов). "Языковая" = обучена предсказывать текст. GPT-4, Claude, LLaMA — LLM. Наш HDC-Brain v14.1 — тоже LLM (маленький, 299M).

Log-Cumsum — логарифм накопленной суммы. В HDCMemory используем для вычисления decay без числового переполнения: log(decay1 × decay2 × ... × decayN) = log(decay1) + log(decay2) + ... + log(decayN). Потом exp() возвращает к нормальным числам.

М

Mass / Масса — в HDCMemory: важность токена для памяти. Число от 0 до 1. mass=1.0 → токен сильно влияет на контекст следующих токенов. mass=0.0 → токен почти игнорируется. Обучается: модель сама решает что важно помнить.

MHA (Multi-Head Attention) — многоголовое внимание. Несколько "голов" смотрят на текст независимо с разных точек зрения. Одна голова может смотреть на синтаксические связи, другая на семантику. Результаты объединяются.

Mixed Precision — смешанная точность. Forward и backward pass в bf16 (быстрее, меньше памяти). Параметры и состояния оптимизатора в float32 (точнее). PyTorch autocast сам выбирает где какой формат.

Н

nn.Embedding — слой PyTorch для хранения векторов токенов. layer = nn.Embedding(32768, 4096) → матрица 32768×4096. layer(token_id) → вектор для этого токена. Обучается как обычные параметры.

nn.Linear — полносвязный слой. Умножает вход на матрицу весов и добавляет смещение: y = x @ W.T + b. Базовый строительный блок. ControllerBlock состоит из двух Linear.

О

Output Scale — масштаб выхода. Обучаемый скаляр на который умножаются логиты. В v14.1 инициализируется как 1.0 (важно! не 1/√dim — это убивает обучение, см. главу 18).

Overfitting / Переобучение — модель "зазубрила" тренировочные данные. Идеально отвечает на виденные примеры, но плохо обобщает. Признак: train loss → 0, val loss остаётся высоким.

П

Parallel Scan — параллельный алгоритм для кумулятивных вычислений. Обычно cumsum = O(T) последовательных операций. Parallel scan = O(log T) параллельных уровней. Используем для HDCMemory чтобы не делать O(T²) attention.

Perplexity / Перплексия — exp(loss). Интерпретация: "сколько вариантов следующего токена рассматривает модель". Perplexity=10 → модель как будто выбирает из 10 равновероятных токенов. Меньше = увереннее = лучше.

Permute / Перестановка — операция HDC. Циклический сдвиг вектора: roll(A, k) сдвигает A на k позиций. Разные k → разные уникальные векторы. Используем для кодирования позиции в Cyclic Position Encoding.

Pre-LN — нормализация ДО подблока. x → LN → sublayer → x + output (а не x → sublayer → x + output → LN). Более стабильные градиенты в глубоких сетях. GPT-2 и наша модель используют Pre-LN.

Р

Residual Connection / Остаточная связь — output = x + block(x). Добавляем вход к выходу блока. Зачем: если блок не знает что делать → пусть просто пропустит вход (block(x) ≈ 0). Градиенты текут через сложение без затухания. Из ResNet (He et al., 2015).

RoPE (Rotary Position Embedding) — позиционное кодирование через поворот. Q и K векторы поворачиваются на угол пропорциональный позиции. Кодирует относительные позиции: "токен A на 3 позиции раньше B". Лучший современный стандарт. Мы используем Cyclic Permute вместо него.

С

Sampling / Семплирование — случайный выбор из распределения. Вместо "всегда берём токен с максимальной вероятностью" (greedy) → случайно выбираем с вероятностями из softmax. Генерация разнообразнее и интереснее.

SentencePiece — библиотека токенизации от Google. Обучает BPE или Unigram словарь на тексте. Работает без пробелов → подходит для русского, японского, финского (где пробелы не так важны). Мы используем готовую модель на 32K токенов.

Sigmoid — функция σ(x) = 1 / (1 + e^(-x)). Превращает любое число в (0, 1). σ(0)=0.5, σ(+∞)=1, σ(-∞)=0. Используем для gate значений в ThoughtLoop и для attention весов в BindingAttention (вместо softmax).

Softmax — превращает вектор в распределение вероятностей (сумма = 1). softmax(x)_i = exp(x_i) / Σ exp(x_j). "Победитель берёт всё": большие значения получают большую вероятность, маленькие — очень маленькую.

STE (Straight-Through Estimator) — трюк для обучения через недифференцируемые операции. sign() имеет нулевой градиент везде → невозможно учить. Трюк: (sign(w) - w).detach() + w. Forward: sign(w). Backward: градиент идёт к w напрямую. Bengio et al. (2013).

Т

Temperature / Температура — параметр генерации. logits_scaled = logits / temperature. T < 1.0 → более уверенная (концентрированная) генерация, ближе к greedy. T > 1.0 → более случайная и разнообразная генерация. T = 1.0 → без изменений.

ThoughtLoop — наш механизм итеративного мышления. 8 блоков применяются 3 раза с гейтированной интерполяцией между проходами. 8 блоков × 3 прохода = 24 "эффективных слоя". Только 24K параметров для управления тремя проходами (gates + thought positions).

Thought Position — обучаемый вектор (4096 чисел) добавляемый ко входу каждого дополнительного прохода ThoughtLoop. Сигнализирует блокам "ты сейчас на второй мысли" или "ты на третьей". Инициализируется с маленьким шумом × 0.01.

Токен — единица текста для модели. Может быть: целое слово ("кот"), часть слова ("косино"), символ ("!"), пробел+слово ("▁кот"). BPE-токенизатор выбирает границы так чтобы частые слова = один токен.

У

Universal Transformer — архитектура (Dehghani et al., 2018) где один блок применяется итеративно (N раз). Idea: переиспользование параметров вместо N разных слоёв. ThoughtLoop развил эту идею добавив learned gates и thought positions.

Ф

Forward Pass — прямой проход. Токены → кодбук → блоки → логиты → loss. Никаких обновлений весов. Просто вычисление. После него идёт backward pass (вычисление градиентов) и шаг оптимизатора.

Х

HDC (Hyperdimensional Computing) — вычислительная парадигма с высокоразмерными векторами. Ключевые свойства: (1) случайные векторы в 4096D почти ортогональны → уникальность; (2) Bind, Bundle, Permute — базовые операции; (3) можно хранить и извлекать информацию.

HDCMemory — наш компонент явной памяти. 8192 параметра (massproj + decayproj). Вычисляет взвешенную накопленную память для каждого токена: context[t] = Σ_{i≤t} decay_factor × mass[i] × x[i]. Параллельно через log-cumsum.

HDCBlock — основной блок v14.1. Три механизма последовательно: HDCMemory (что было?) → MultiHeadBindingAttention (на что смотреть?) → ControllerBlock (подумать). ~21M параметров, из которых 99.9% — в ControllerBlock.

Ц

Cyclic Permutation — наш метод позиционного кодирования. Токен на позиции t: циклически сдвигаем его вектор на t элементов. Вектор "Кот" на позиции 0 ≠ "Кот" на позиции 5. 0 параметров. Работает для любой длины последовательности.

Э

Embedding — вектор который представляет дискретный объект (токен). "Кот" → вектор из 4096 чисел. Похожие слова → похожие векторы (в хорошей модели). Основа всех языковых моделей.

Я

Языковая модель — модель которая учится предсказывать следующий токен. Обучение: предсказываем токен, сравниваем с правильным, обновляем веса. После достаточного обучения модель "понимает" язык — и может генерировать, переводить, отвечать на вопросы.
