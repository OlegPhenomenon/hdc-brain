Доказательства: почему 3 мысли работают

"Это должно работать" — не аргумент.

Любой инженер скажет: докажи. Покажи данные. Объясни почему.

В этой главе — именно это. Эксперименты, числа, гипотезы.

Эксперимент: одна модель, три настройки

Мы обучили одну и ту же архитектуру в трёх режимах:

  

    CODE
    
  

  Настройка A: 1 проход через блоки (n_thoughts = 1)
Настройка B: 2 прохода           (n_thoughts = 2)
Настройка C: 3 прохода           (n_thoughts = 3)

Всё остальное одинаково: те же 8 блоков, те же параметры, те же данные, тот же batch. Единственная разница — сколько раз прогоняем через блоки.

    CODE

  7.8 |× A (1 мысль)
      |  ×
  7.4 |    ×  × B (2 мысли)
      |          ×
  6.0 |              ×  × C (3 мысли)
      |                       ×
  4.5 |                             ×  ← только C добирается сюда
      |
      +──────────────────────────────→
      100  500  1000  1500  2000

Конкретные числа:

  

    CODE
    
  

  Шаг 500:
  1 мысль:  loss = 7.64
  2 мысли:  loss = 7.43
  3 мысли:  loss = 6.91   ← уже в 1.1× лучше

Шаг 1000:
  1 мысль:  loss = 7.51   ← начинает стагнировать

После 2000 шагов разрыв огромный: 7.49 vs 4.52.

BPB: что значат эти числа

После 40,000 шагов обучения:

  

    CODE
    
  

  1 мысль:  BPB ≈ 6.8   — модель почти не понимает язык
2 мысли:  BPB ≈ 6.4   — немного лучше, но застряла
3 мысли:  BPB ≈ 4.2   — приемлемое качество!

Что такое BPB? Это "Bits Per Byte" — сколько бит нужно чтобы закодировать один байт текста, если использовать нашу модель как компрессор.

Ориентиры:

  

    CODE
    
  

  BPB = 1.0  — идеальное предсказание (недостижимо)
BPB = 2.0  — очень хорошо (GPT-4 уровень для английского)
BPB = 4.0  — хорошо (маленькие модели)
BPB = 6.0  — плохо, модель плохо понимает язык
BPB = 7.5  — почти случайное угадывание

При 3 мыслях мы добиваемся BPB 4.2 — это разумный результат для 299M-параметровой модели. При 1 мысли — 6.8 — это почти "не работает".

Как меняются gates во время обучения

Следим за значениями thought_gates (через sigmoid) в процессе обучения v14.1:

  

    CODE
    
  

                      gate_2    gate_3
Инициализация:      0.500     0.500    ← нейтрально (sigmoid(0) = 0.5)
Шаг 100:            0.501     0.499    ← почти нет движения
Шаг 500:            0.512     0.491    ← gate_2 начинает открываться
Шаг 1000:           0.524     0.488    ← gate_2 растёт, gate_3 чуть падает
Шаг 1650:           0.531     0.488    ← gate_2 = 53%, gate_3 = 49%

Что это означает?

  • gate_2 открывается быстро: вторая мысль полезна, модель это "поняла"
  • gate_3 открывается медленно: третья мысль — тонкая настройка, нужна не всегда

Это похоже на:

  

    CODE
    
  

  Мысль 1: "прочитать" — обязательна
Мысль 2: "понять глубже" — открывается быстро, явно нужна
Мысль 3: "финальная проверка" — медленнее открывается, тонкая корректировка

Почему 2 мысли — ловушка плато

Самый загадочный результат: почему 2 недостаточно?

Объяснение 1: Числа слоёв

  

    CODE
    
  

  SmolLM2 (360M):     32 слоя, 1 проход  = 32 "шага обработки"
HDC-Brain, 3 мысли:  8 блоков × 3     = 24 "шага обработки"
HDC-Brain, 2 мысли:  8 блоков × 2     = 16 "шага обработки"  ← мало!

16 шагов — это уровень GPT-2 Small (12 слоёв) по глубине. Недостаточно для хорошего понимания языка.

Объяснение 2: Симметрия системы

С двумя мыслями:

  

    CODE
    
  

  h = (1-gate₂) × h₁ + gate₂ × h₂

Есть только одна точка обратной связи. Система легко находит "удобное равновесие" где gate₂ = 0.5 и обе мысли вносят одинаковый вклад — но это не оптимум.

С тремя мыслями:

  

    CODE
    
  

  h' = (1-gate₂) × h₁ + gate₂ × h₂
h'' = (1-gate₃) × h' + gate₃ × h₃

Две точки обратной связи. Система не может "застрять" так же легко. Между двумя точками можно установить "направление движения" и стабилизироваться.

Объяснение 3: Три уровня абстракции

  

    CODE
    
  

  После мысли 1:
  h₁ = Blocks(tokens)
  ← "поверхностный уровень": синтаксис, базовые связи

После мысли 2:
  h₂ = Blocks(h₁)
  ← "семантический уровень": значения, тематические группы

После мысли 3:
  h₃ = Blocks(h')  где h' — смесь h₁ и h₂
  ← "прагматический уровень": контекст, предсказание

Третья мысль обрабатывает уже обогащённый результат. Как третье прочтение книги — когда уже знаешь сюжет и видишь тонкие детали.

Четыре мысли: убывающая отдача

Мы попробовали n_thoughts = 4:

  

    CODE
    
  

  3 мысли:  BPB = 4.2,  время на шаг = 1.0× (базис)
4 мысли:  BPB = 3.9,  время на шаг = 1.33×

Улучшение: +7% за 33% больше времени

Есть улучшение, но небольшое.

Это классическая "убывающая отдача":

  

    CODE
    
  

  Мысль 1: +огромный прирост (с нуля до базового понимания)
Мысль 2: +большой прирост  (от поверхностного к глубокому)
Мысль 3: +значительный прирост
Мысль 4: +небольшой прирост
Мысль 5: +минимальный прирост

Каждый дополнительный проход добавляет меньше предыдущего.

Мы зафиксировали TRAIN_THOUGHTS = 3 как оптимальный баланс.

Thought positions: зачем они нужны

Вернёмся к thought_pos — обучаемым векторам для каждой мысли.

Доказательство что они важны:

  

    CODE
    
  

  Обучение с thought_pos:    BPB = 4.2
Обучение без thought_pos:  BPB = 4.7  (≈ 12% хуже)

Почему?

Без thought_pos блоки "не знают" на каком проходе они работают. Первый и третий проход — одинаковый входной сигнал (только h меняется). Блоки не могут адаптировать своё поведение.

С thought_pos — каждый проход начинается с другим дополнительным вектором:

  

    CODE
    
  

  Проход 2: h + thought_pos[1]  ← "добавляем сигнал мысли 2"
Проход 3: h + thought_pos[2]  ← "добавляем сигнал мысли 3"

Блоки учатся: "когда вижу сигнал мысли 2 — работаю как 'аналитик'. Когда вижу сигнал мысли 3 — работаю как 'финализатор'."

Сравнение с Universal Transformer

Идея взята из Universal Transformer (Dehghani et al., 2018).

Сравнение:

  

    CODE
    
  

  Universal Transformer (UT)       HDC Thought Loops
───────────────────────────────────────────────────
Один универсальный блок          8 специализированных блоков
Adaptive Computation Time        Фиксированное 3 прохода
Прямое h = новый результат       Гейтированная интерполяция
Временная позиция (число шага)   Thought-позиция (вектор)

Наши улучшения:

  1. 8 блоков вместо 1 — каждый проход более мощный
  2. Learned gates — контролируемое смешивание, не прямая замена
  3. Thought positions — семантика прохода, не просто номер

Gradient Checkpointing и три мысли

Технический момент, важный для понимания.

С 3 мыслями — 24 прохода через блоки (8 блоков × 3). При backpropagation PyTorch нужны промежуточные активации.

Без оптимизации:

  

    CODE
    
  

  Память = 24 прохода × batch × tokens × dim × bf16
       = 24 × 12 × 512 × 4096 × 2 байта
       ≈ 6 ГБ только активации!

С gradient checkpointing:

  

    CODE
    
  

  Память ≈ 750 МБ  (в 8 раз меньше)
Цена: вычисляем каждый блок дважды (forward + пересчёт при backprop)

Без checkpointing — 3 мысли просто не влезут в 24 ГБ GPU.

Итоговая картина

Почему именно 3 мысли, именно 8 блоков?

  

    CODE
    
  

  8 блоков × 1 проход:  168M параметров, 8 "шагов" — мало глубины
8 блоков × 3 прохода: 168M параметров, 24 "шага" — оптимально
8 блоков × 4 прохода: 168M параметров, 32 "шага" — чуть лучше, но дороже

Мысли переиспользуют параметры:
  Без Thought Loops: нужно 168M × 3 = 504M для той же глубины
  С Thought Loops:   168M параметров при 3× вычислительной глубине

Thought positions:  16K параметров → +12% качества
Learned gates:       4 параметра   → контролируемое смешивание
