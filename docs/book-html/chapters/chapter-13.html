Экспедиция IV Глава 13

Attention — самый важный механизм в NLP

Коля — следователь.

К нему приходит свидетель и рассказывает историю. Коля слушает — но не все слова запоминает одинаково. Имена — запоминает хорошо. "Синяя машина" — запоминает. "Погода была неплохой" — пропускает мимо.

Когда потом нужно понять "кто это сделал?" — Коля обращается к нужным деталям, а не ко всей истории целиком.

Это и есть внимание (attention).

Проблема HDCMemory

В главе 12 мы узнали про HDCMemory.

HDCMemory хорошо запоминает весь контекст — взвешенно, с затуханием. Но она "размывает" всё вместе. Все прошлые токены складываются в один вектор.

Пример:

  

    CODE
    
  

  "The animal didn't cross the street because it was tired"
   0      1        2      3     4       5       6   7    8

Токен "it" (позиция 6) должен понять: "it" = "animal" (уставшее животное, не улица).

HDCMemory запомнит и "animal", и "street" примерно одинаково — оба в прошлом. Она не умеет выбрать конкретный токен и сфокусироваться на нём.

Нужен другой механизм — который умеет целиться в нужный токен.

Идея: поиск в архиве

Представь систему поиска документов.

У тебя есть запрос: "французская революция 1789 год"

У каждого документа есть метка (что в нём): "история Франции", "экономика XIX века", "рецепты"...

Ты сравниваешь запрос с каждой меткой. Находишь подходящие документы. Берёшь содержимое этих документов.

В Attention:

  • Query (Q) — "запрос" текущего токена: "что я ищу?"
  • Key (K) — "метка" каждого токена: "о чём я?"
  • Value (V) — "содержимое" каждого токена: "что во мне?"

Токен "it" спрашивает: "где тут одушевлённое существо?" Токен "animal" отвечает: "я о одушевлённом существе" (высокий score). Токен "street" молчит: "я о неодушевлённом" (низкий score).

Математика Attention: шаг за шагом

Допустим у нас предложение из T токенов, каждый — вектор размерности D.

Шаг 1: Три проекции — Q, K, V

  

    PYTHON
    
  

  import torch
import torch.nn as nn
import math

W_q = nn.Linear(D, d_k)  # матрица для запросов
W_k = nn.Linear(D, d_k)  # матрица для ключей
W_v = nn.Linear(D, d_v)  # матрица для значений

# x: (T, D) — наши токены
Q = W_q(x)   # (T, d_k) — запросы
K = W_k(x)   # (T, d_k) — ключи
V = W_v(x)   # (T, d_v) — значения

Зачем три разные матрицы?

Каждая создаёт "свой взгляд" на токен:

  • Q — "что я ищу в других токенах?"
  • K — "какую информацию я могу предоставить другим?"
  • V — "что я реально отдам, если меня выбрали?"

Шаг 2: Похожесть запроса с каждым ключом

  

    PYTHON
    
  

  scores = Q @ K.transpose(-2, -1)  # (T, T)
# scores[i, j] = Q[i] · K[j]
# = насколько запрос токена i совпадает с ключом токена j

Высокий score = токен i "заинтересован" в токене j.

Шаг 3: Масштабирование

  

    PYTHON
    
  

  scores = scores / math.sqrt(d_k)

Зачем делить на √d_k?

Вспомни главу 11 (output_scale): скалярное произведение двух векторов растёт пропорционально √D. Без масштабирования — scores были бы огромными, softmax выбирал бы один токен с 99.9% вероятностью. Делим чтобы нормализовать.

Шаг 4: Causal mask — не смотрим в будущее

  

    PYTHON
    
  

  mask = torch.tril(torch.ones(T, T)).bool()  # нижнетреугольная матрица
scores = scores.masked_fill(~mask, float('-inf'))
# Токен на позиции i не может видеть токены j > i

Почему это важно? Наша модель обучается предсказывать следующий токен. Если токен i "видит" токен i+1 — это жульничество. Ответ уже известен.

Шаг 5: Softmax → вероятности

  

    PYTHON
    
  

  attn_weights = F.softmax(scores, dim=-1)  # (T, T)
# attn_weights[i] — вектор от 0 до 1, сумма = 1.0
# Показывает на кого "смотрит" токен i

Шаг 6: Взвешенная сумма значений

  

    PYTHON
    
  

  output = attn_weights @ V  # (T, d_v)
# output[i] = взвешенная комбинация V-векторов всех предыдущих токенов

Полный код одного Attention

  

    PYTHON
    
  

  import torch
import torch.nn.functional as F
import math

def attention(Q, K, V, causal=True):
    """
    Q, K: (B, T, d_k)
    V:    (B, T, d_v)
    """
    d_k = Q.shape[-1]

    # Похожесть
    scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)  # (B, T, T)

    # Causal mask
    if causal:
        T = Q.shape[-2]
        mask = torch.tril(torch.ones(T, T, device=Q.device)).bool()
        scores = scores.masked_fill(~mask, float('-inf'))

    # Вероятности
    weights = F.softmax(scores, dim=-1)

    # Взвешенная сумма
    return weights @ V   # (B, T, d_v)

Пример: как "it" находит "animal"

Представим что модель уже обучена. Смотрим на веса внимания токена "it":

  

    CODE
    
  

  "The animal didn't cross the street because it was tired"
  0     1       2      3     4       5       6   7    8

Q[6] = вектор-запрос для "it"

Scores токена "it" (позиция 6):
  scores[6, 0] = Q[6] · K[0]   ("The")     ≈  0.1
  scores[6, 1] = Q[6] · K[1]   ("animal")  ≈  2.1  ← высокий!
  scores[6, 2] = Q[6] · K[2]   ("didn't")  ≈  0.2
  scores[6, 3] = Q[6] · K[3]   ("cross")   ≈  0.5
  scores[6, 4] = Q[6] · K[4]   ("the")     ≈  0.1
  scores[6, 5] = Q[6] · K[5]   ("street")  ≈  0.8

После softmax:
  weights[6] = [0.02, 0.58, 0.03, 0.06, 0.02, 0.10, ...]
                      ↑
               58% внимания на "animal"!

output[6] ≈ 0.58 × V["animal"] + 0.10 × V["street"] + ...
          ≈ вектор который несёт информацию о "animal"

Модель "поняла" что "it" скорее всего = "animal" — через высокое внимание.

Multi-Head Attention: несколько точек зрения

Один механизм внимания смотрит на текст с одной точки зрения. Но нам нужно несколько:

  • Голова 1: кто с кем грамматически связан (подлежащее-сказуемое)
  • Голова 2: местоимения и их референты (it → animal)
  • Голова 3: близкие слова (соседние позиции)
  • Голова 4: тематические связи (все слова про Францию)

Multi-Head Attention (MHA) — запускаем несколько attention-голов параллельно:

  

    PYTHON
    
  

  class MultiHeadAttention(nn.Module):
    def __init__(self, dim, n_heads):
        super().__init__()
        self.n_heads = n_heads
        self.head_dim = dim // n_heads     # размерность одной головы

        # Одна большая матрица для всех голов сразу
        self.W_q = nn.Linear(dim, dim)
        self.W_k = nn.Linear(dim, dim)
        self.W_v = nn.Linear(dim, dim)
        self.W_o = nn.Linear(dim, dim)    # финальная проекция

    def forward(self, x):
        B, T, D = x.shape
        H = self.n_heads
        HD = self.head_dim  # D // H

        # Проецируем и разбиваем на H голов
        Q = self.W_q(x).view(B, T, H, HD).transpose(1, 2)  # (B, H, T, HD)
        K = self.W_k(x).view(B, T, H, HD).transpose(1, 2)  # (B, H, T, HD)
        V = self.W_v(x).view(B, T, H, HD).transpose(1, 2)  # (B, H, T, HD)

        # Attention для каждой головы (параллельно)
        scores = Q @ K.transpose(-2, -1) / math.sqrt(HD)   # (B, H, T, T)
        mask = torch.tril(torch.ones(T, T, device=x.device)).bool()
        scores = scores.masked_fill(~mask, float('-inf'))
        weights = F.softmax(scores, dim=-1)                # (B, H, T, T)
        out = weights @ V                                   # (B, H, T, HD)

        # Объединяем H голов обратно в размерность D
        out = out.transpose(1, 2).contiguous().view(B, T, D)  # (B, T, D)

        return self.W_o(out)  # финальная проекция

Сколько параметров стоит Attention

Для нашей размерности (D = 4096, H = 4):

  

    CODE
    
  

  W_q: 4096 × 4096 = 16,777,216 параметров
W_k: 4096 × 4096 = 16,777,216 параметров
W_v: 4096 × 4096 = 16,777,216 параметров
W_o: 4096 × 4096 = 16,777,216 параметров
─────────────────────────────────────────
ИТОГО: 67,108,864 ≈ 67M параметров на один слой

Если у нас 8 слоёв: 67M × 8 = 536M параметров только на attention.

Это в 1.8 раза больше чем вся наша модель (299M)!

Вот почему мы не используем стандартный Attention. Нам нужно что-то умнее.

Softmax: "победитель берёт всё"

Обратим внимание на softmax в attention.

Softmax "заостряет" распределение: большие значения становятся ещё больше.

  

    PYTHON
    
  

  import torch

scores = torch.tensor([0.1, 2.0, 0.3, 0.5])
weights = F.softmax(scores, dim=-1)
print(weights)
# tensor([0.057, 0.760, 0.070, 0.113])
#                ↑
#         76% внимания на один элемент!

Это хорошо для чётких связей ("it" → "animal"). Плохо для нечётких (когда нужно учитывать несколько токенов равномерно).

Есть вариант — использовать sigmoid вместо softmax. Sigmoid не требует чтобы сумма весов была 1.0. Каждый вес независим. Наша архитектура использует именно это в следующей главе.

Формула Attention

  

    CODE
    
  

  Attention(Q, K, V) = softmax(Q × K^T / √d_k) × V

где:
  Q ∈ R^(T × d_k) — запросы
  K ∈ R^(T × d_k) — ключи
  V ∈ R^(T × d_v) — значения
  d_k — размерность ключей (для масштабирования)

Вот что каждый элемент означает буквально:

  • Q × K^T — матрица похожести: каждый запрос сравнивается с каждым ключом
  • / √d_k — нормализация: убираем эффект роста значений с размерностью
  • softmax(...) — превращаем похожести в вероятности (сумма = 1)
  • × V — взвешиваем содержимое: "возьми 58% от animal, 10% от street..."

Attention — механизм "избирательного внимания":

  

    CODE
    
  

  Q (query)  — "что я ищу?"
K (key)    — "что я предлагаю для поиска?"
V (value)  — "что я отдаю, если меня выбрали?"

Attention(Q, K, V) = softmax(Q × K^T / √d) × V

Шаги:

  1. Проецируем токены в Q, K, V через обучаемые матрицы
  2. Считаем похожесть: scores = Q @ K^T / √d
  3. Causal mask: будущее → -∞
  4. Softmax → вероятности (сумма = 1)
  5. Взвешенная сумма V-векторов

Multi-Head: несколько голов = несколько точек зрения.
