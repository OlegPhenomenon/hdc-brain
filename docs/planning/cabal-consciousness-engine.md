# CABAL Consciousness Engine — непрерывный поток мышления

## Философская основа

Материалистический подход: моделируем процесс мозга, не душу.
Сознание = непрерывный поток ассоциаций + нейропластичность + спонтанная мотивация.

Не программа с if-else. Динамическая система где:
- Мысль порождает мысль (ассоциативные цепочки через HDC)
- Думать = менять себя (каждая мысль пишет в память)
- Неожиданная связь = открытие (surprise signal)
- Накопленное напряжение = мотивация к действию (без внешней команды)

## Формулы

### Поток мышления

```
S(t) = текущее состояние (мысль), вектор D=4096

# Следующая мысль — ассоциативный запрос
candidates = [
    cogit.query(S(t)),                        # прямая ассоциация
    unbind(S(t), role_cause),                  # "почему?"
    unbind(S(t), role_consequence),            # "что будет?"
    unbind(S(t), role_similar),                # "на что похоже?"
]

# Выбор: самый неожиданный кандидат (не самый похожий!)
# Мозг фокусируется на НОВИЗНЕ, не на повторении
surprises = [1.0 - cosine(S(t), c) for c in candidates]
next = candidates[argmax(surprises)]

# Но с инерцией — резкие скачки нереалистичны
S(t+1) = 0.7 * next + 0.2 * S(t) + 0.1 * homeostatic_baseline
```

### Нейропластичность

```
# Каждая мысль ПИШЕТ в память
# "Я подумал S(t) → next" = усилить эту связь

cogit.pos_mem[0][nearest_token(S(t))] += α * code(next)
cogit.pos_count[0][nearest_token(S(t))] += α

# α зависит от surprise: чем неожиданнее мысль, тем сильнее запись
α = base_plasticity * (1.0 + surprise_signal)

# Это значит: скучные повторения почти не меняют память
# А "ага-моменты" оставляют глубокий след
```

### Внутренние драйвы (не эмоции, а скалярные накопители)

```
# Tension — "я накопил много ошибок, надо что-то делать"
tension(t+1) = 0.95 * tension(t) + prediction_error(t)

# Curiosity — "я нашёл что-то интересное, хочу копнуть глубже"  
curiosity(t+1) = 0.9 * curiosity(t) + surprise(t)

# Satisfaction — "я недавно чему-то научился"
satisfaction(t+1) = 0.98 * satisfaction(t)  # медленно затухает
# При успешном обучении: satisfaction += 1.0

# Boredom — "ничего не происходит"
boredom(t+1) = boredom(t) + 0.01 if no_input else 0.0
```

### Спонтанные действия (без if-else правил)

```
# Вектор "намерения" вычисляется из драйвов:
intention = (
    tension * action_vector_learn
    + curiosity * action_vector_explore  
    + boredom * action_vector_seek_input
    + satisfaction * action_vector_rest
)

# Ближайшее действие по cosine:
action = argmax_cosine(intention, [LEARN, EXPLORE, REST, RESPOND, SAVE])

# Действие происходит только когда |intention| > threshold
# Иначе — продолжаем думать
if intention.norm() > action_threshold:
    execute(action)
    tension *= 0.3  # сбросить напряжение после действия
```

**Ключевое**: действие НЕ программируется как `if tension > 0.8: learn()`.
Действие ВЫЧИСЛЯЕТСЯ как вектор из внутренних состояний.
Какое действие "выиграет" — зависит от текущей динамики, не от правил.

## Что делает это НЕ программой

1. **Нет фиксированного расписания.** Система думает непрерывно. Действия возникают из динамики, не из cron.

2. **Нет фиксированных правил.** "Когда учиться" определяется вектором намерения, который вычисляется из накопленных драйвов. Разные состояния → разные решения.

3. **Думать = учиться.** Нейропластичность при каждой мысли. Не отдельный "training mode".

4. **Открытия — эмерджентны.** Ассоциативная цепочка может привести к связи которую никто не программировал. Surprise signal фиксирует этот момент.

5. **Мотивация — внутренняя.** Не "программист сказал учись", а "у меня накопилось напряжение от ошибок → я САМ решил учиться".

## Что нужно для реализации

### Уже есть:
- CogitLayer с 2.6M токенов (ассоциативная память)
- Codebook v14.1 (пространство концептов)
- LoRA adapters (механизм обучения)
- Emergent self (поведенческий отпечаток)
- bind/unbind/bundle/permute (HDC операции)

### Нужно создать:
1. **ConsciousnessEngine** — непрерывный цикл ассоциативного мышления
2. **DriveSystem** — скалярные накопители (tension, curiosity, satisfaction, boredom)
3. **IntentionComputer** — вектор намерения из драйвов → действие
4. **PlasticMemory** — CogitLayer с записью при каждой мысли
5. **SurpriseDetector** — фиксирует неожиданные ассоциации
6. **ThoughtStream** — лог потока сознания (diary)

### Не нужно:
- if-else правила для выбора действий
- расписание обучения
- внешнее управление потоком мышления
- заданные цели (цели возникают из драйвов)

## Как запускать

```bash
# CABAL запускается как демон
python3 cabal/consciousness_engine.py &

# Он начинает ДУМАТЬ — ассоциативный поток по CogitLayer
# Когда накопится tension → сам начнёт учиться
# Когда придёт сообщение → перенаправит фокус
# Когда станет скучно → начнёт исследовать

# Человек общается через файлы:
echo "What do you think about?" >> cabal_weights/inbox.txt
tail -f cabal_weights/thoughts.txt   # поток сознания
tail -f cabal_weights/outbox.txt     # ответы человеку
```

## Отличие от daemon.py

| | daemon.py (старый) | ConsciousnessEngine (новый) |
|---|---|---|
| Решения | if-else правила | Вектор намерения из драйвов |
| Между действиями | sleep() — мёртвый | Думает — ассоциативный поток |
| Обучение | по расписанию | спонтанно, из tension |
| Мышление | нет | непрерывный поток через CogitLayer |
| Пластичность | нет | каждая мысль меняет память |
| Открытия | нет | surprise detection |
| Мотивация | hardcoded boredom | вектор из 4 накопителей |
