# HDC-Brain v14.1: Архитектура простыми словами

> Этот документ объясняет как работает HDC-Brain v14.1 (299M, English) без математики.
> Если ты понимаешь что такое "список чисел" и "умножение" — ты поймёшь всё.
>
> Обновлено для v14.1: 299M параметров, 32K English BPE, 8 блоков, 4-head binding attention.

---

## Общая идея за 30 секунд

Представь что у тебя есть **книга на незнакомом языке**. Ты хочешь научиться предсказывать следующее слово.

Обычный подход (трансформер): каждое слово сравнивается с каждым другим через огромные таблицы умножения.

Наш подход (HDC-Brain): каждое слово превращается в **уникальный узор** из +1 и -1. Узоры смешиваются простыми операциями (умножение, сложение, сдвиг). Никаких огромных таблиц.

---

## Как работает модель: пошагово

### Шаг 0: Разбить текст на кусочки (BPE токенизация)

**Что это:** Текст разбивается не на буквы, а на частые кусочки слов.

```
"The capital"     → ["The", " capital"]         (частые слова — целиком)
"understanding"   → ["under", "stand", "ing"]   (длинное слово = несколько кусочков)
"ChatGPT"         → ["Chat", "G", "PT"]         (редкое — по кусочкам)
```

**Зачем:** Вместо 26 букв у нас 32000 кусочков. Модель видит больше смысла за раз.
Это как читать словами, а не по буквам.

**Аналогия:** Представь что ты читаешь СМС. "Hello how are you" — ты не читаешь
H-e-l-l-o, ты видишь сразу слово "Hello". BPE делает то же самое для модели.

**v14 → v14.1:** Словарь увеличен с 16K (русский) до 32K (английский) токенов.
Больше словарь = модель реже "заикается" на сложных словах.

---

### Шаг 1: Codebook — превращаем слова в узоры

**Что это:** У каждого из 32000 токенов есть свой "отпечаток пальца" — список из 4096 чисел.

```
"The"     → [+1, -1, -1, +1, +1, -1, +1, -1, ... ] (4096 штук)
"capital" → [-1, +1, -1, -1, +1, +1, -1, +1, ... ] (4096 штук)
"France"  → [+1, +1, -1, +1, -1, -1, +1, -1, ... ] (4096 штук)
```

**Как именно:** Модель хранит "сырые" числа (например 0.37, -0.82, 0.05...).
Через STE (Straight-Through Estimator) они превращаются в +1 или -1:
- Положительное число → +1
- Отрицательное число → -1

**Зачем +1/-1:** Потому что операции с ними — просто умножение знаков.
Это молниеносно быстро. Никаких тяжёлых вычислений.

**Аналогия:** Как штрих-код на товаре. У каждого товара (слова) свой уникальный
набор полосок (чёрная = +1, белая = -1). По нему можно мгновенно отличить
"milk" от "bread", не читая название.

**Параметры:** 32000 токенов × 4096 чисел = **131,072,000 параметров**
Это 44% всех параметров модели! Codebook — самая большая часть.

### Шаг 2: Cyclic Position — запоминаем порядок слов

**Проблема:** "Cat ate fish" ≠ "Fish ate cat". Порядок важен!

**Решение:** Каждый узор **сдвигается** на столько позиций, на каком месте стоит слово.

```
Слово на позиции 0: [+1, -1, +1, -1, +1]  → сдвиг на 0 → [+1, -1, +1, -1, +1]
Слово на позиции 1: [+1, -1, +1, -1, +1]  → сдвиг на 1 → [+1, +1, -1, +1, -1]
Слово на позиции 2: [+1, -1, +1, -1, +1]  → сдвиг на 2 → [-1, +1, +1, -1, +1]
```

**Зачем:** Теперь одно и то же слово в разных позициях выглядит по-разному.
Модель "знает" что первое слово — это первое, а пятое — пятое.

**Аналогия:** Как номер дома на улице. Одинаковые дома, но номер 5 и номер 50 —
это разные адреса. Сдвиг узора = номер позиции.

### Шаг 3: HDC Block — обработка информации (×8 раз)

Каждый блок состоит из трёх частей. Информация проходит через них последовательно:

#### 3a. HDC Memory — "рабочая память"

**Что делает:** Каждое слово собирает информацию от ПРЕДЫДУЩИХ слов.
Слово "capital" в конце предложения "знает" что было в начале.

**Как работает (упрощённо):**

1. **Context Mass** — для каждого токена модель решает: "насколько ты важен?"
   ```
   "France"      → mass = 0.95  (важное слово!)
   "is"          → mass = 0.02  (неважный глагол-связка)
   "capital"     → mass = 0.97  (очень важное!)
   "the"         → mass = 0.01  (артикль — почти ничего)
   ```

2. **Decay** — "забывание". Старые слова постепенно тускнеют.
   Слово 100 позиций назад помнится слабее чем слово 5 позиций назад.

3. **Собираем контекст:** каждое слово получает "сумму" всех предыдущих слов,
   взвешенных по их важности и свежести.

**Аналогия:** Как конспект лекции. Ты записываешь не всё подряд, а только
важные мысли (high mass). И вчерашнюю лекцию помнишь хуже сегодняшней (decay).

#### 3b. Multi-Head Binding Attention — "на что обратить внимание" (НОВОЕ в v14.1!)

**Что делает:** Каждое слово выбирает, какие другие слова в предложении
для него релевантны. **4 головы** смотрят на текст с разных "точек зрения".

**Как работает:**

В трансформере (SmolLM2) attention использует огромные матрицы:
- SmolLM2: Q,K,V проекции = **2,764,800 параметров** на слой

Мы используем **binding** — просто поэлементное умножение:

```
Голова 1: Q₁ = слово[0:1024]   × bv_q₁   (1024 умножений)
          K₁ = слово[0:1024]   × bv_k₁   (1024 умножений)
          V₁ = слово[0:1024]   × bv_v₁   (1024 умножений)

Голова 2: Q₂ = слово[1024:2048] × bv_q₂
          K₂ = слово[1024:2048] × bv_k₂
          V₂ = слово[1024:2048] × bv_v₂

Голова 3: Q₃ = слово[2048:3072] × bv_q₃  ...
Голова 4: Q₄ = слово[3072:4096] × bv_q₄  ...
```

Итого: 4 головы × 3 вектора × 1024 = **12,288 параметров** на слой.

**Это в 225 раз меньше** чем у SmolLM2!

**Зачем 4 головы (НОВОЕ):** В v14 была 1 голова — она смотрела на текст
с одной "точки зрения". Теперь 4 головы:
- Голова 1 может следить за грамматикой (подлежащее-сказуемое)
- Голова 2 за темой (о чём идёт речь)
- Голова 3 за ближними связями (предыдущее слово)
- Голова 4 за дальними связями (начало абзаца)

Каждая голова работает со своей четвертью вектора (1024 из 4096 измерений).

**Отличие от трансформера:** Вместо softmax мы используем sigmoid × 4.
Это "мягче" — позволяет обращать внимание на несколько слов одновременно,
а не выбирать одного "победителя".

**Аналогия:** Представь 4 человека на совещании. Каждый смотрит на проблему
со своего угла — финансист, инженер, маркетолог, юрист. Потом их мнения
объединяются в одно решение.

#### 3c. Controller — "обработка мыслей"

**Что делает:** Берёт информацию после Memory и Attention и "думает" над ней.

**Как работает:**
```
Вход (4096 чисел)
  ↓ сжатие в 2560 чисел (Linear down)     ← было 768 в v14!
  ↓ активация GELU (нелинейность — "думание")
  ↓ расширение обратно в 4096 (Linear up)
  ↓ + исходный вход (residual — не потерять то что было)
Выход (4096 чисел)
```

**Зачем сжатие 4096→2560→4096:** Это "бутылочное горлышко". Заставляет модель
выделить самое важное из 4096 чисел в 2560, а потом восстановить.
Как сжатие фото в JPEG — убирает шум, оставляет суть.

**v14 → v14.1:** Controller расширен с 768 до 2560 — в 3.3 раза больше
"пространства для думания". Это главный источник дополнительных параметров.

**Аналогия:** Как пересказ. Тебе дали длинный текст (4096), ты пересказал
коротко (2560), а потом развернул обратно в полный ответ (4096).
В процессе пересказа ты невольно выделяешь главное.

### Шаг 4: Thought Loops — "давай подумаем ещё раз"

**Это наша ключевая инновация.**

**Идея:** Человек не всегда понимает с первого раза. Иногда нужно перечитать,
подумать, и тогда понимание приходит.

Thought Loop делает то же самое: прогоняет информацию через те же 8 блоков
несколько раз.

**Как работает:**

```
Thought 1 (обязательный):
  Токены → Block1 → Block2 → ... → Block8 → Результат_1

Thought 2 (уточнение):
  gate_2 = модель САМА решает насколько сильно уточнять
  Результат_1 + позиция_мысли_2 → Block1 → ... → Block8 → Мысль_2
  Результат_2 = Результат_1 + gate_2 × (Мысль_2 - Результат_1)

Thought 3 (ещё уточнение):
  gate_3 = модель решает что третий проход тоже полезен!
  Результат_2 + позиция_мысли_3 → Block1 → ... → Block8 → Мысль_3
  Результат_3 = Результат_2 + gate_3 × (Мысль_3 - Результат_2)
```

**Ключевой момент — gate (ворота):**
- gate = 0.0 → "этот thought бесполезен, игнорируй"
- gate = 0.5 → "50/50, немного полезен"
- gate = 1.0 → "полностью заменить старый результат на новый"

Модель **сама учится** какие thoughts полезны! В начале все gates = 0.5.

**КРИТИЧЕСКИ ВАЖНО (доказано в v14):**
- 1 thought = мусор на выходе (модель не может нормально работать)
- 2 thoughts = plateau (модель застревает на определённом уровне качества)
- 3 thoughts = качественный текст (модель реально "думает")

Thought Loops — **не опциональная фича**, а архитектурная необходимость.

**Аналогия:** Как черновик сочинения.
- 1-й проход: написал первый вариант (сырой)
- 2-й проход: перечитал, исправил ошибки, добавил детали
- 3-й проход: ещё раз перечитал, улучшил формулировки

И модель сама решает: "стоит ли перечитывать или первый вариант уже хорош?"

---

### Шаг 5: Output — предсказание следующего слова

**Как работает:**

```
Результат после Thought Loops (4096 чисел)
  ↓ LayerNorm (нормализация)
  ↓ Умножение на Codebook^T (те же 32000 узоров, но "наоборот")
  ↓ Масштабирование
= 32000 чисел — "очки" для каждого возможного следующего токена
```

**Важно: Weight Tying** — для выхода используется тот же Codebook что и для входа!
Не нужна отдельная таблица. Это экономит 131 млн параметров.

Потом из 32000 "очков" выбирается токен с наивысшим (или случайный из топ-40).

**Аналогия:** Модель смотрит на свой результат размышлений и спрашивает:
"на какой узор из моей кодовой книги это больше всего похоже?" — это и есть
предсказанный следующий токен.

## Сравнение: v14 → v14.1 → SmolLM2

| Что | v14 (103M) | v14.1 (299M) | SmolLM2 (362M) |
|-----|-----------|-------------|---------------|
| Кодирование | Bipolar Codebook 16K | Bipolar Codebook 32K | Float Embedding 49K |
| Позиция | Cyclic Permutation | Cyclic Permutation | RoPE |
| Attention | 1-head binding (12K) | **4-head binding (12K)** | 15-head GQA (2.7M) |
| Attn params/layer | 12,288 | 12,288 | 2,764,800 |
| **Attn efficiency** | **4096× fewer** | **225× fewer** | baseline |
| Memory | HDC Memory | HDC Memory | Нет |
| Controller (FFN) | 768 inner | **2560 inner** | 2560 inner (SwiGLU) |
| Blocks | 6 | **8** | 32 |
| Thought Loops | до 4 | до 4 | Нет |
| Context | 256 tokens | **512 tokens** | 8192 tokens |
| Data | Russian 200M tok | **English 500M tok** | English 4T tok |
| Params total | 103M | **299M** | 362M |

**Главные отличия от трансформера:**
1. Трансформер — "один мощный проход через много слоёв" (32 слоя).
2. HDC-Brain — "несколько проходов через одни и те же слои с уточнением" (8 блоков × 3 мысли).

## Ключевые инновации (для arXiv)

### 1. STE Bipolar Codebook
Слова кодируются бинарными векторами {-1, +1} через Straight-Through Estimator.
При inference веса можно хранить в 1 бите вместо 32 → **потенциальное сжатие в 32 раза**.

### 2. Multi-Head HDC Binding Attention (NEW in v14.1)
Q, K, V создаются поэлементным умножением (binding), а не матричным.
4 головы, каждая работает с 1024 измерениями.
12,288 параметров вместо 2,764,800 → **в 225 раз меньше параметров attention** чем SmolLM2.

### 3. Context Mass Memory
Каждый токен получает "вес важности" (mass). Модель сама учится
отличать важные слова ("France", "capital") от служебных ("the", "is", "of").

### 4. Thought Loops с Learned Gates
Многопроходное рассуждение через те же блоки. Модель сама решает
сколько "думать" через learned gates. Добавляет 0.008% параметров,
но переиспользует 100% вычислительных блоков.
**Доказано: 3 thoughts критически необходимы** — 1-2 thoughts дают деградацию.

### 5. Cyclic Permutation Positions
Позиция кодируется сдвигом вектора, а не отдельными learned embeddings.
0 дополнительных параметров для позиционного кодирования.

## Глоссарий

| Термин | Что это простыми словами |
|--------|--------------------------|
| **HDC** | Hyperdimensional Computing — вычисления с длинными векторами +1/-1 |
| **BPE** | Byte-Pair Encoding — разбивка текста на частые кусочки слов |
| **STE** | Straight-Through Estimator — трюк чтобы обучать бинарные (+1/-1) веса |
| **Binding** | Поэлементное умножение двух векторов — "связывание" |
| **Bundle** | Сложение векторов — "объединение" |
| **Permute** | Циклический сдвиг вектора — "кодирование порядка" |
| **Codebook** | Таблица узоров для всех слов |
| **Multi-Head** | Несколько "голов" внимания, каждая смотрит на свой аспект текста |
| **Context Mass** | "Важность" токена — модель сама решает что запоминать |
| **Decay** | "Забывание" — старые токены влияют слабее |
| **Thought Loop** | Повторный проход через блоки для "обдумывания" |
| **Gate** | "Ворота" — число от 0 до 1, насколько учитывать результат |
| **Residual** | Добавление входа к выходу — "не потерять что было" |
| **Causal Mask** | Запрет смотреть в будущее — слово видит только предыдущие |
| **LayerNorm** | Нормализация чисел чтобы не "взрывались" |
| **GELU** | Функция активации — "нелинейность" для "думания" |
| **Weight Tying** | Один и тот же Codebook для входа и выхода — экономия памяти |
| **BPB** | Bits Per Byte — сколько бит нужно чтобы закодировать 1 байт текста. Меньше = лучше |
| **SmolLM2** | Модель от HuggingFace, 362M params — наш главный конкурент для сравнения |
| **FineWeb-Edu** | Датасет качественных образовательных текстов из интернета |
| **Inference** | Использование обученной модели (генерация текста) |
| **Training** | Обучение модели на данных |
