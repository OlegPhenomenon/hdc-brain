## История первая: Маша обучает нейросеть

Маша — разработчик в стартапе. Её задача: сделать систему которая умеет:
1. Распознавать котов на фотографиях
2. Распознавать собак на фотографиях

Казалось бы просто. Она обучает нейросеть на котах — точность 99%. Отлично!

Теперь добавляет собак. Обучает...

```
До обучения на собаках:
  Точность котов: 99%
  Точность собак: 0%

После обучения на собаках:
  Точность котов: 43%   ← УПАЛА!
  Точность собак: 97%
```

Маша в панике. Почему котов стало 43%? Она же ничего не трогала!

---

### Почему это происходит

Нейросеть — это миллионы чисел (весов). Когда она "учит котов", эти числа меняются так чтобы распознавать котов. Когда потом "учит собак" — *те же самые числа* меняются снова.

```
Нейросеть обучается на котах:
  Веса = [..., 0.7, -0.3, 0.9, 0.1, ...]   ← "знания о котах" закодированы здесь

Потом обучается на собаках:
  Веса = [..., 0.2, 0.8, -0.1, 0.6, ...]   ← "знания о собаках" перезаписали!
             ↑
     Числа изменились — и знания о котах потерялись
```

Это называется **catastrophic forgetting** — "катастрофическое забывание".

> **Catastrophic forgetting** (произносится "кэтастрофик форгеттинг")
> = когда нейросеть учит новое и при этом забывает старое.
> "Катастрофическое" потому что забывание полное и внезапное.

---

### Твой мозг не забывает

А вот твой мозг так не работает. Ты умеешь распознавать котов — и не забудешь это после того как узнаешь как выглядят собаки. Ты умеешь ездить на велосипеде — и не разучишься плавать.

В мозге разные воспоминания хранятся в разных местах. Они не перезаписывают друг друга.

Именно это наблюдение привело к созданию HDC.

## История вторая: Пентти Каэнерва и адреса в огромном городе

1988 год. Финский учёный Пентти Каэнерва (произносится "Кэнерва") работает в NASA и думает о памяти.

> Нейробиологи знали: каждое воспоминание хранится *распределённо* —
> не в одном нейроне, а в тысячах.
> И мозг устойчив к повреждениям: если убить часть нейронов,
> воспоминания не исчезают полностью — они "размыты" и сохранились.

Каэнерва придумал модель памяти которая работает так же. Ключевая идея:

**Информацию нужно кодировать в очень длинных случайных векторах.**

Зачем *очень длинных*? Сейчас объясним. Это одна из самых красивых идей в математике.

## Магия высоких измерений

### Задача: как не мешать друг другу

Представь огромный город. В нём 10 миллионов жителей — и у каждого должен быть уникальный адрес. Если адреса короткие (например, "дом 7, квартира 3"), они быстро закончатся.

Но если адреса длинные и случайные (например, "дом АГ39Ж, корпус Б7Р, квартира 000471К...") — адреса никогда не закончатся, и два случайных адреса никогда не совпадут.

Это и есть идея: **в пространстве большой размерности случайные векторы почти никогда не похожи друг на друга**.

---

### Эксперимент: насколько похожи случайные векторы

```python
import torch

def average_similarity(dim, n_pairs=5000):
    """
    Создаём n_pairs пар случайных векторов.
    Считаем среднее косинусное сходство.

    Косинусное сходство:
    +1.0 = одинаковые
     0.0 = перпендикулярные (ортогональные, не похожи вообще)
    -1.0 = противоположные
    """
    total = 0
    for _ in range(n_pairs):
        a = torch.randn(dim)
        b = torch.randn(dim)
        # Формула косинусного сходства:
        # cos(a, b) = (a · b) / (|a| × |b|)
        cos = torch.dot(a, b) / (a.norm() * b.norm())
        total += abs(cos.item())  # берём абсолютное значение
    return total / n_pairs


print("Размерность → среднее сходство случайных векторов:")
for dim in [2, 10, 100, 1000, 4096]:
    sim = average_similarity(dim)
    bar = "█" * int(sim * 50)  # визуализация
    print(f"  dim={dim:>5}: {bar:<25} {sim:.4f}")
```

```
Размерность → среднее сходство случайных векторов:
  dim=    2: ████████████████████     0.418  ← очень похожи!
  dim=   10: ████████                 0.176
  dim=  100: ██                       0.056
  dim= 1000:                          0.018
  dim= 4096:                          0.009  ← почти нулевое!
```

**В 4096 измерениях случайные векторы практически перпендикулярны.** Их сходство 0.009 — это почти 90 градусов.

Как это понять? Представь компас. Когда у тебя только 2 направления — почти любые две стрелки будут куда-то "смотреть рядом". Когда у тебя 4096 направлений — две случайные стрелки смотрят в совершенно разные стороны.

---

### Что это нам даёт

Наш словарь — 32,000 токенов. Каждому нужен уникальный вектор. В 4096 измерениях все они будут практически ортогональными — "не мешают" друг другу:

```
"кот"  → [ +1, -1, +1, -1, +1, ...]  (4096 чисел)
"пёс"  → [ -1, +1, -1, +1, -1, ...]
"дом"  → [ +1, +1, -1, -1, +1, ...]
"мир"  → [ -1, -1, +1, +1, -1, ...]
...

Сходство "кот" и "пёс": 0.009  ← почти нулевое
Сходство "кот" и "кот": 1.000  ← точно одинаковые (само с собой)
```

Это значит: каждое слово получает *уникальное пространство* в векторе. Разные слова не "мешают" друг другу. Добавляешь новое слово — не перезаписываешь старые.

**Это и есть решение проблемы Маши.** В HDC каждый факт хранится в своём "адресе". Новые факты не перезаписывают старые.

> Научное название для этого: **квазиортогональность**.
> "Квази" = "почти". "Ортогональность" = перпендикулярность.
> Квазиортогональность = почти перпендикулярные, почти не пересекаются.
> Не пугайся слова — ты уже понял суть из примеров выше.

---

## Зачем это называется "Гиперпространственные Вычисления"

**Hyperdimensional Computing** = **HDC** — дословно "вычисления в гиперпространстве".

"Гиперпространство" — это математический термин для пространств с очень большим числом измерений. Не космос из фантастики, а просто "больше трёх измерений".

- 2D: плоскость (лист бумаги)
- 3D: наш обычный мир
- 4096D: гиперпространство, которое невозможно визуализировать, но с ним можно работать математически

В этом гиперпространстве:
- Каждое слово = точка (вектор)
- Похожие слова = близкие точки (высокое косинусное сходство)
- Разные слова = далёкие точки (сходство ≈ 0)
- Операции над векторами = "вычисления над смыслами"

---

## Устойчивость к ошибкам: бонус который не ожидали

HDC-векторы имеют ещё одно свойство — устойчивость к случайным ошибкам.

**Эксперимент**: возьми вектор слова "кот" (4096 элементов). Случайно поменяй 10% элементов на противоположные. Насколько результат ещё похож на оригинал?

```python
def add_noise(vector, noise_level=0.1):
    """
    Случайно переворачиваем noise_level долю элементов.
    Например, noise_level=0.1 = 10% элементов перевернём.
    """
    n_flip = int(len(vector) * noise_level)
    indices = torch.randperm(len(vector))[:n_flip]
    noisy = vector.clone()
    noisy[indices] = -noisy[indices]  # +1 становится -1 и наоборот
    return noisy


original = torch.sign(torch.randn(4096))  # случайный вектор "кот"

print("Шум → сходство с оригиналом:")
for noise_pct in [0.01, 0.05, 0.10, 0.20, 0.40]:
    noisy = add_noise(original, noise_pct)
    cos = torch.dot(original, noisy) / (original.norm() * noisy.norm())
    bar = "█" * int(cos.item() * 20)
    print(f"  {noise_pct*100:4.0f}% шума: {bar:<20} {cos:.3f}")
```

```
Шум → сходство с оригиналом:
    1% шума: ████████████████████ 0.980  ← почти идеально
    5% шума: ████████████████     0.900
   10% шума: ████████████████     0.800  ← 80% смысла сохранено!
   20% шума: ████████████         0.600
   40% шума: ████                 0.200
```

10% ошибок → 80% сходства! Это значит: даже если вычисления дали небольшие ошибки (что бывает в реальных GPU) — вектор всё ещё "узнаваем".

В нейросети 10% шума в активациях может полностью разрушить предсказание. В HDC — нет.

## HDC vs нейросети: принципиальная разница

Не просто "ещё один подход". Фундаментально другой принцип.

```
ОБЫЧНАЯ НЕЙРОСЕТЬ (трансформер):
────────────────────────────────
  Знания → распределены по ВСЕМ весам
         → новые знания могут перезаписать старые
         → нужно обучаться на всех данных одновременно
         → матрицы весов — "чёрный ящик", непонятно что там

HDC:
────────────────────────────────
  Знания → каждый факт = один вектор
         → факты складываются (Bundle), не мешая друг другу
         → можно добавлять новые факты без переобучения
         → операции понятны и интерпретируемы
```

---

### Аналогия которую легко запомнить

**Нейросеть** — это человек который выучил 100 стихотворений наизусть. Он умеет их говорить, но объяснить *правила* стихосложения не может. И если учит 101-е стихотворение — может начать путать 100-е.

**HDC** — это человек который изучил правила стихосложения. Размер, рифма, ударения — всё по правилам. Он может применить правила к любому новому стихотворению, не забывая предыдущих.

---

## Где уже используется HDC прямо сейчас

HDC — не академическая игрушка. Несколько реальных применений:

### Умные часы (классификация жестов)

```
Задача: распознать жест рукой (взмах, тап, поворот)
Ресурсы: ARM Cortex-M4 процессор, 256KB памяти

Нейросеть: не влезает — слишком тяжёлая
HDC: работает! Операции с векторами очень дёшевые
Точность: сравнима с нейросетью, скорость в 100× выше
```

### Анализ ДНК

```
Задача: к какому виду относится этот фрагмент ДНК?
Данные: миллиарды пар оснований

HDC подход: каждый k-мер (кусочек ДНК) кодируется как вектор,
           все векторы объединяются через Bundle,
           классификация за один проход
Результат: быстрее BLAST (стандарт индустрии)
```

### HDC-Brain (наш проект)

```
Задача: языковая модель (предсказание следующего токена)
Новизна: HDC не как отдельный классификатор,
         а как ФУНДАМЕНТ языковой модели

Токены = HDC-векторы
Attention = HDC-операции (Bind вместо матриц)
Позиции = HDC-Permute (0 параметров)
```

## Карта идей: как HDC встроен в нашу архитектуру

Прежде чем идти глубже — посмотрим на общую схему.

```
Стандартный трансформер → HDC-Brain: что изменилось

┌────────────────────────────────────────────────────────────────┐
│ Компонент          │ Трансформер              │ HDC-Brain      │
├────────────────────┼──────────────────────────┼────────────────┤
│ Токен → вектор     │ Embedding (float)         │ Bipolar ±1     │
│                    │ 32M параметров            │ 32M параметров │
│                    │                           │ но ±1, не float│
├────────────────────┼──────────────────────────┼────────────────┤
│ Позиция токена     │ RoPE (обучаемое)          │ Cyclic Permute │
│                    │ D/2 параметров            │ 0 параметров!  │
├────────────────────┼──────────────────────────┼────────────────┤
│ Query, Key, Value  │ Linear(D, D) × 3          │ Bind (вектор D)│
│                    │ 3 × D² параметров         │ 3 × D параметров│
│                    │ = 50 млн каждая           │ = 4,096 каждая │
├────────────────────┼──────────────────────────┼────────────────┤
│ Комбинировать      │ Concat + Linear(D,D)      │ Просто Concat  │
│ головы внимания    │ D² параметров             │ 0 параметров!  │
├────────────────────┼──────────────────────────┼────────────────┤
│ Embedding ↔ выход  │ Два разных словаря        │ Один словарь   │
│                    │ 2 × 32K × D               │ (weight tying) │
│                    │ = 131M дополнительных     │ экономия 131M  │
└────────────────────┴──────────────────────────┴────────────────┘
```

Каждое изменение — это применение HDC-принципа к конкретной задаче. В следующих главах разберём каждый блок.

## Итоги главы

- **Catastrophic forgetting** — главная проблема нейросетей: новые знания перезаписывают старые
- **Каэнерва (1988)** предложил хранить информацию в длинных случайных векторах
- **Квазиортогональность**: в 4096D можно иметь 32,000+ практически уникальных векторов
- **Устойчивость к ошибкам**: 10% шума → 80% сходства (нейросети так не умеют)
- **HDC в нашей архитектуре**: каждый компонент заменён HDC-операцией, экономим сотни миллионов параметров

Но пока мы не видели *как именно* работают HDC-операции. Это следующая глава — самая важная в книге.
