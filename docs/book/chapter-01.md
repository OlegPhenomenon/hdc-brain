Ты только что сделал то, что языковая модель делает миллионы раз в секунду.

## Одно предложение которое объясняет всё

> **Языковая модель** — это программа которая для любого текста отвечает на один вопрос: "какое слово идёт следующим и с какой вероятностью?"

Если бы это была Ruby-программа, её интерфейс выглядел бы так:

```ruby
# Представь что это метод в Rails-контроллере
def predict_next(text)
  # text = "Столица Франции это"
  # возвращает хеш: {слово => вероятность}
  {
    "Париж"  => 0.87,
    "город"  => 0.06,
    "страна" => 0.02,
    "крупный"=> 0.01,
    # ... остальные 32000 слов → ~0%
  }
end
```

Вот и всё. Один метод. Один вопрос. Один ответ: распределение вероятностей по всем словам в словаре.

## Запусти прямо сейчас: твоя первая языковая модель

Прежде чем читать дальше — запусти это. Не нужно понимать каждую строку. Просто запусти и посмотри что происходит.

Создай файл `first_lm.py`:

```python
text = """
кот сидит на коврике кот ест рыбу кот спит на диване
собака бежит по парку собака лает на кота собака ест кость
кот и собака живут в доме кот боится собаку
"""

words = text.lower().split()

# Для каждого слова запоминаем: что идёт после него?
model = {}
for i in range(len(words) - 1):
    word = words[i]
    next_word = words[i + 1]
    if word not in model:
        model[word] = {}
    model[word][next_word] = model[word].get(next_word, 0) + 1

# Спрашиваем модель: что идёт после "кот"?
print("После 'кот':")
after = model["кот"]
total = sum(after.values())
for word, count in sorted(after.items(), key=lambda x: -x[1]):
    print(f"  {word:12s} → {count/total:5.0%}")

# А после "собака"?
print("\nПосле 'собака':")
after = model["собака"]
total = sum(after.values())
for word, count in sorted(after.items(), key=lambda x: -x[1]):
    print(f"  {word:12s} → {count/total:5.0%}")
```

Запусти: `python3 first_lm.py`

Ты увидишь что-то вроде:

```
После 'кот':
  сидит        →   20%
  ест          →   20%
  спит         →   20%
  и            →   20%
  боится       →   20%

После 'собака':
  бежит        →   33%
  лает         →   33%
  ест          →   33%
```

Поздравляю — ты только что построил свою первую языковую модель. Словарь `model` — это она и есть. Для каждого слова он хранит вероятности следующего.

Это называется **биграмная модель** (bigram model). "Би" = два, "грам" = элемент. Она смотрит на пары: текущее слово → следующее слово.

## Генерация: одно слово за другим

Теперь добавь в конец `first_lm.py`:

```python
import random

def generate(model, start, length=10):
    result = [start]
    current = start
    for _ in range(length):
        if current not in model:
            break
        options = model[current]
        total = sum(options.values())
        # Выбираем случайно, но с учётом частот
        r = random.random() * total
        cumulative = 0
        for word, count in options.items():
            cumulative += count
            if r <= cumulative:
                result.append(word)
                current = word
                break
    return " ".join(result)

print("\n--- Генерация ---")
for _ in range(5):
    print(generate(model, "кот", 8))
```

Запусти ещё раз. Каждый раз — другой текст:

```
кот ест рыбу кот спит на диване собака
кот сидит на коврике кот ест рыбу кот
кот боится собаку собака лает на кота собака
```

**ChatGPT работает ТОЧНО ТАК ЖЕ.** Серьёзно. Он предсказывает следующее слово, добавляет его к тексту, предсказывает следующее. И так снова и снова.

Разница? Масштаб:

```
Твоя модель:   ~30 слов,      смотрит на 1 предыдущее слово
ChatGPT:       весь интернет,  смотрит на 128 000 предыдущих слов
```

Но принцип — один и тот же.

## Почему наша модель плохая

Запусти генерацию 20 раз. Заметишь проблему: текст зацикливается.

```
кот сидит на коврике кот сидит на коврике кот
```

Почему? Наша модель видит только **одно предыдущее слово**. После "коврике" всегда идёт "кот". После "кот" — случайный вариант. Но после "сидит" — всегда "на". Цикл.

Модель не знает что предложение уже закончилось. Не видит дальний контекст.

Это как если бы ты в Rails писал:

```ruby
# Плохо — решение на основе одного поля
def recommend(last_viewed_product)
  Product.where(category: last_viewed_product.category).sample
end

# Хорошо — учитываем всю историю
def recommend(user)
  viewed = user.viewed_products.last(100)
  # анализируем паттерны по ВСЕЙ истории
end
```

Нейронные языковые модели решают эту проблему: они видят **весь контекст сразу** — не одно предыдущее слово, а сотни тысяч.

## Клавиатура телефона vs ChatGPT

Подсказки на клавиатуре телефона — тоже языковая модель. Но маленькая:

```
┌──────────────────┬─────────────────┬──────────────────┐
│                  │ Клавиатура      │ ChatGPT          │
├──────────────────┼─────────────────┼──────────────────┤
│ Принцип          │ след. слово     │ след. слово      │
│ Параметров       │ ~10 000         │ ~1 000 000 000 000│
│ Данные           │ твои SMS        │ весь интернет    │
│ Контекст         │ 3 слова         │ 128 000 слов     │
│ Помнит диалог    │ нет             │ да               │
└──────────────────┴─────────────────┴──────────────────┘
```

**Параметры** — это "настройки" модели. Числа внутри которые определяют поведение. Как `config/credentials.yml` в Rails-проекте — набор значений которые определяют как система работает. Только в нейросетях этих значений миллиарды и подбираются они автоматически.

**Контекст** — сколько предыдущих слов модель "видит" при предсказании. Клавиатура видит 3. GPT-4 видит 128 тысяч — весь разговор целиком.

Наша модель, **HDC-Brain v14.1**, — это 299 миллионов параметров. В 3000 раз меньше чем GPT-4, но в 30 000 раз больше клавиатуры. И устроена принципиально иначе — но об этом в Экспедиции II.

## Токены — не слова, а кусочки

В коде выше мы разбивали текст по пробелам — получали слова. Настоящие языковые модели делают иначе.

Проблема: что делать с редким словом которое модель никогда не видела?

```
"антидисэстаблишментаризм" → ??? (нет в словаре!)
```

Решение — разбить на **популярные кусочки**:

```
"антидисэстаблишментаризм"
  → ["анти", "дис", "эста", "блиш", "мент", "аризм"]
```

Каждый кусочек встречается часто. Модель их знает. Вместе — дают смысл.

Эти кусочки называются **токены**. Алгоритм который их создаёт — **BPE**.

**BPE** расшифровывается как **Byte-Pair Encoding** — "кодирование байтовыми парами." Он работает как zip-архиватор: находит часто повторяющиеся пары символов и заменяет их одним токеном. Самые частые пары сливаются первыми. Повторяем пока не получим словарь нужного размера.

Подробно разберём в Главе 9. Сейчас запомни: модель работает не со словами, а с токенами — кусочками от одной буквы до целого слова.

## Что значит "обучить" модель

Наша биграмная модель обучалась просто: считали частоты. Нейронные модели обучаются иначе.

Берём текст и делаем из него задачи:

```
Текст: "кот сидит на коврике"

Задача 1:  вход = ["кот"]                → правильный ответ: "сидит"
Задача 2:  вход = ["кот", "сидит"]       → правильный ответ: "на"
Задача 3:  вход = ["кот", "сидит", "на"] → правильный ответ: "коврике"
```

На каждой задаче:
1. Модель делает предсказание (вероятности для каждого слова)
2. Сравниваем с правильным ответом — считаем ошибку
3. Чуть-чуть подкручиваем параметры чтобы ошибка уменьшилась
4. Повторяем

Это как `rails db:seed` + тестирование в цикле: засеиваем данные, проверяем результат, фиксим код, повторяем. Только вместо "фиксить код" — автоматически подкручиваются миллионы чисел внутри модели.

В реальности — миллиарды задач, сотни миллионов параметров, тысячи часов GPU. Но идея — та же что в нашем `first_lm.py`: посмотри на текст, запомни паттерны.

## Почему предсказание = понимание

Стоп. Предсказание следующего слова — это ведь просто статистика. Как из этого получается "понимание"?

Посмотри на эти примеры:

```
"Он открыл ___ и выпил воду"
    → "кран" или "бутылку"

"Он открыл ___ и начал читать"
    → "книгу" или "журнал"

"Врач сказал что операция ___"
    → "прошла успешно"

"Программист сказал что операция ___"
    → "занимает O(n²)"
```

Чтобы правильно заполнить пропуск, модель должна "знать":
- что "кран" связан с "водой", а "книга" — с "чтением"
- что у слова "операция" разный смысл для врача и программиста
- что O(n²) — это из мира программирования

Это и есть понимание. Оно возникает **автоматически** — как побочный эффект хорошего предсказания. Никто не учил GPT-4 что такое метафора или ирония. Он научился сам — потому что без этого нельзя точно предсказать следующее слово в тексте.

## Наш путь — другой

Все большие языковые модели — GPT-4, LLaMA, Mistral — построены на архитектуре **трансформер** (transformer). Её придумали в 2017 году, и с тех пор альтернатив почти не было.

Наша модель, HDC-Brain v14.1, построена **не на трансформерах**. Она использует другую математику — **Hyperdimensional Computing** (вычисления в гиперпространстве). Что это и зачем — начнём разбирать в Экспедиции II.

А пока — фундамент. Следующая глава: как компьютер вообще "видит" слова.
