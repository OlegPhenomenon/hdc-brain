### Пациент и три слова

Возьмём конкретный пример. Входная строка:

```
"Кот сидел на"
```

Модель должна предсказать следующее слово. Может быть "коврике", "крыше", "диване"...

Проследим каждый шаг. Размерности будем писать явно, как штамп.

### Шаг 1: Токенизация — слова в числа

```python
import sentencepiece as spm

tokenizer = spm.SentencePieceProcessor("tokenizer.model")
text = "Кот сидел на"
tokens = tokenizer.encode(text)
print(tokens)  # → [7234, 15893, 422]
```

Три слова — три числа. Каждое число от 0 до 32767 (наш словарь = 32К токенов).

Что означают числа:
- `7234` = токен "Кот"
- `15893` = токен "▁сидел" (▁ = пробел перед словом)
- `422` = токен "▁на"

Частые слова → один токен. Редкие слова → несколько токенов:
```
"кот" → один токен
"нейровычисления" → три токена: ["▁ней", "ро", "вычисления"]
```

Форма на этом этапе:
```
tokens: [7234, 15893, 422]  — одномерный список из 3 чисел
```

### Шаг 2: Кодбук — числа в векторы

Числа сами по себе бессмысленны (422 > 7234 — это не значит что "на" > "Кот"). Нужно превратить их в векторы.

```python
# Извлекаем строки из кодбука:
raw = model.codebook.weight[tokens]  # форма: (3, 4096)
# raw[0] = вектор для "Кот"   (4096 вещественных чисел)
# raw[1] = вектор для "сидел" (4096 вещественных чисел)
# raw[2] = вектор для "на"    (4096 вещественных чисел)

# Применяем STE (бинаризация):
x = torch.sign(raw.detach()) * raw.detach() / raw.detach() + raw - raw.detach()
# Упрощённо: x ≈ sign(raw) ∈ {-1, +1}
```

После этого каждый токен — биполярный вектор:

```
x[0] = [+1, -1, +1, +1, -1, ..., +1]  ← вектор "Кот", 4096 чисел {-1,+1}
x[1] = [-1, +1, -1, +1, +1, ..., -1]  ← вектор "сидел"
x[2] = [+1, +1, -1, -1, +1, ..., +1]  ← вектор "на"
```

Квазиортогональность: любые два вектора из 32К — почти перпендикулярны.

Форма: `(1, 3, 4096)` — добавили размерность батча (B=1).

### Шаг 3: Позиционное кодирование — где какое слово

Проблема: "Кот сидел на" и "сидел Кот на" — разные векторы у токенов одинаковые!

```python
x = model._cyclic_position(x)
```

Внутри:

```python
for t in range(T):  # для каждой позиции
    shift = t % hdc_dim  # позиция 0→0, 1→1, ..., 4095→4095, 4096→0
    indices = (torch.arange(hdc_dim) - shift) % hdc_dim
    x[:, t, :] = x[:, t, :].gather(-1, indices.expand_as(x[:, t, :]))
```

Что происходит:
- Позиция 0 ("Кот"): сдвиг = 0, вектор не меняется
- Позиция 1 ("сидел"): сдвиг = 1, вектор циклически сдвигается на 1
- Позиция 2 ("на"): сдвиг = 2, вектор циклически сдвигается на 2

Теперь "Кот" на позиции 0 и "Кот" на позиции 5 — **разные** векторы.

Форма: `(1, 3, 4096)` — не изменилась, параметров не добавилось.

### Шаг 4-6: ThoughtLoop — три прохода через 8 блоков

Это главная часть. Три итерации, каждая — полный проход через все 8 блоков.

#### Мысль 1: базовый анализ

```python
h = x  # входной тензор: (1, 3, 4096)

for i, block in enumerate(model.blocks):  # 8 блоков
    h = block(h)

h1 = h  # результат первой мысли: (1, 3, 4096)
```

Внутри каждого блока — три механизма. Разберём на примере одного блока:

**4а. HDCMemory: что было раньше**

```python
# Внутри block.memory(h):
mass  = torch.sigmoid(self.mass_proj(h.norm(dim=-1, keepdim=True)))
# mass: (1, 3, 1) — насколько важен каждый из 3 токенов
# mass[0,0] ≈ 0.6 ("Кот" важен)
# mass[0,1] ≈ 0.7 ("сидел" немного важнее)
# mass[0,2] ≈ 0.5 ("на" менее важно)

decay = torch.sigmoid(self.decay_proj(h.norm(dim=-1, keepdim=True)))
# decay: (1, 3, 1) — насколько быстро этот токен "затухает" в памяти

weighted = mass * h
# weighted: (1, 3, 4096) — взвешенные токены

# Параллельный скан (log-cumsum): строим накопленную память
context = parallel_scan(weighted, decay)
# context[0, 2, :] = накопленная память на позиции 2 ("на"):
#   = затухший_Кот + затухший_сидел

mem_output = context  # (1, 3, 4096)
```

После residual + LayerNorm:
```python
h = self.ln_mem(h + mem_output)  # (1, 3, 4096)
```

**4б. MultiHeadBindingAttention: на что обратить внимание**

```python
# Внутри block.attention(h):
# 4 головы, каждая голова смотрит с dim=1024

q = self.bv_q * h.reshape(B, T, H, D//H)  # broadcasting
k = self.bv_k * h.reshape(B, T, H, D//H)
# bv_q и bv_k — маленькие обучаемые векторы {-1,+1} (4×1024)

# HDC Bind = поэлементное умножение:
binding = q * k  # (1, 4, 3, 1024) — НЕТ матрицы T×T!

# Bundle по измерениям (суммирование):
scores = binding.sum(-1)  # (1, 4, 3) — оценки для каждого токена

# Causal mask: "на" не смотрит на будущее
mask = torch.tril(torch.ones(T, T))  # (3, 3) — нижнетреугольная

# Sigmoid (не softmax!) × 4 для масштаба:
attn_weights = torch.sigmoid(scores) * mask[:T] * 4  # (1, 4, 3)

# Применяем к значениям:
v = h
attended = attn_weights.unsqueeze(-1) * v.reshape(B, T, H, D//H)
attn_output = attended.reshape(B, T, D)  # (1, 3, 4096)
```

После residual + LayerNorm:
```python
h = self.ln_attn(h + attn_output)  # (1, 3, 4096)
```

**4в. ControllerBlock: глубокая обработка**

```python
# Внутри block.controller(h):
# Pre-LN → expand → GELU → contract → residual

h_normed = self.ln(h)                    # нормализуем: (1, 3, 4096)
hidden = self.fc_down(h_normed)          # 4096→2560: (1, 3, 2560)
hidden = F.gelu(hidden)                  # нелинейность
h_out = self.fc_up(hidden)               # 2560→4096: (1, 3, 4096)
h = h + self.ln2(h_out)                  # residual: (1, 3, 4096)
```

ControllerBlock содержит 21M параметров — 99.9% одного блока.

После всех 8 блоков: `h1 = (1, 3, 4096)`.

---

#### Мысль 2: уточнение

```python
gate_2 = torch.sigmoid(model.thought_loop.thought_gates[1])
# gate_2 ≈ 0.53 (открылось за обучение)

# Подготовка входа: нормализуем h1 + добавляем "метку мысли 2"
thought_input = model.thought_loop.ln(h1) + model.thought_loop.thought_pos[1]
# thought_pos[1]: обучаемый вектор (4096) — "ты на второй мысли"

# Снова через ТЕ ЖЕ 8 блоков:
thought = thought_input
for block in model.blocks:
    thought = block(thought)
h2_raw = thought  # (1, 3, 4096)

# Гейтированная интерполяция:
h2 = h1 + gate_2 * (h2_raw - h1)
# = (1 - 0.53) × h1  +  0.53 × h2_raw
# = 0.47 × h1 + 0.53 × h2_raw
```

Смысл: не заменяем h1 полностью — берём взвешенную смесь. Если gate_2=0, первая мысль остаётся. Если gate_2=1, вторая мысль полностью заменяет.

Форма: `h2 = (1, 3, 4096)`.

---

#### Мысль 3: финализация

```python
gate_3 = torch.sigmoid(model.thought_loop.thought_gates[2])
# gate_3 ≈ 0.49

thought_input = model.thought_loop.ln(h2) + model.thought_loop.thought_pos[2]
# thought_pos[2]: "ты на третьей мысли, финализируй"

thought = thought_input
for block in model.blocks:
    thought = block(thought)
h3_raw = thought

# Финальная интерполяция:
h_final = h2 + gate_3 * (h3_raw - h2)
# ≈ 0.51 × h2 + 0.49 × h3_raw
```

После трёх мыслей: `h_final = (1, 3, 4096)`.

Три прохода через 8 блоков = 24 "эффективных слоя". Тот же h_final несёт информацию о "Кот", "сидел", "на" — со всем контекстом и тремя уровнями анализа.

### Шаг 7: Декодирование — векторы обратно в слова

```python
# Умножаем на транспонированный кодбук:
# h_final: (1, 3, 4096)
# codebook.weight.T: (4096, 32768)
# Результат: (1, 3, 32768)

logits = (h_final @ model.codebook.weight.T) * model.output_scale
# logits[0, 2, :] = (32768,) — оценки для каждого возможного следующего токена
#                               ПОСЛЕ позиции 2 ("на")
```

Важная деталь: `model.output_scale` — обучаемый параметр, инициализирован как 1.0.

Зачем weight_tying? Тот же кодбук на входе и выходе:
- Вход: `codebook[token_id]` → вектор токена
- Выход: `h_final @ codebook.T` → схожесть финального вектора с каждым токеном

Логичная симметрия: если "коврике" и наш финальный вектор близки в HDC-пространстве → высокий score для "коврике".

### Шаг 8: Softmax и выбор следующего слова

```python
# Берём последнюю позицию (предсказание после "на"):
next_logits = logits[0, -1, :]  # (32768,)

# Вероятности:
probs = torch.softmax(next_logits, dim=-1)  # сумма = 1.0

# Топ-5:
top_values, top_indices = torch.topk(probs, k=5)
for p, idx in zip(top_values, top_indices):
    word = tokenizer.decode([idx.item()])
    print(f"  {word:15s} {p.item()*100:.1f}%")

# Пример вывода (после достаточного обучения):
#   ▁коврике        31.2%
#   ▁крыше          18.7%
#   ▁диване         12.4%
#   ▁полу            9.1%
#   ▁подоконнике     7.3%
```

Температура — регулятор "уверенности":

```python
# Холодная температура (0.5): модель более уверена, топ-токены вероятнее
probs_cold = torch.softmax(next_logits / 0.5, dim=-1)

# Горячая температура (1.5): более случайные ответы, меньше повторений
probs_hot  = torch.softmax(next_logits / 1.5, dim=-1)
```

Семплирование (не greedy-выбор):

```python
next_token_id = torch.multinomial(probs, num_samples=1).item()
next_word = tokenizer.decode([next_token_id])
# → "коврике" (в 31% случаев) или другое
```

context_ids = tokenizer.encode("Кот сидел")

### Все размерности одним взглядом

Вот как меняется форма тензора на каждом шаге:

```
"Кот сидел на"          строка
   ↓ токенизация
[7234, 15893, 422]       список из 3 чисел
   ↓ + батч-размерность
(1, 3)                   B=1, T=3
   ↓ кодбук lookup
(1, 3, 4096)             каждый токен → 4096 чисел {-1,+1}
   ↓ cyclic position
(1, 3, 4096)             форма не меняется! (только числа внутри)
   ↓ ThoughtLoop мысль 1
(1, 3, 4096)             форма не меняется!
   ↓ ThoughtLoop мысль 2
(1, 3, 4096)             форма не меняется!
   ↓ ThoughtLoop мысль 3
(1, 3, 4096)             форма не меняется!
   ↓ @ codebook.T
(1, 3, 32768)            каждая позиция → оценки для 32K токенов
   ↓ берём последнюю позицию
(32768,)                 одномерный вектор вероятностей
   ↓ softmax + sample
1                        один токен-ответ
   ↓ decode
"коврике"                строка
```

**Ключевой момент**: на протяжении почти всей обработки форма остаётся `(B, T, 4096)`. Модель работает с постоянной размерностью, только преобразуя содержимое.

### Во время обучения vs при генерации

Есть важное отличие:

**Обучение (тренировка):**

```python
# Входная последовательность: "Кот сидел на коврике"
# Токены: [7234, 15893, 422, 8831]

x = tokens[:-1]  # [7234, 15893, 422]   — вход (без последнего)
y = tokens[1:]   # [15893, 422, 8831]   — цель (сдвинуто на 1)

logits = model(x)  # (1, 3, 32768)

# Считаем loss для КАЖДОЙ позиции одновременно:
# Позиция 0: x="Кот",   предсказываем y="сидел"
# Позиция 1: x="сидел", предсказываем y="на"
# Позиция 2: x="на",    предсказываем y="коврике"

loss = cross_entropy(logits.reshape(-1, vocab_size), y.reshape(-1))
```

Всё считается параллельно! Один forward pass — три предсказания.

**Генерация (inference):**

```python
# Один шаг = одно новое слово
context = [7234, 15893, 422]  # "Кот сидел на"
logits = model(torch.tensor([context]))
next_token = sample(logits[0, -1, :])  # берём ТОЛЬКО последнюю позицию
context.append(next_token)
# → [7234, 15893, 422, 8831]  # "Кот сидел на коврике"
```

Во время генерации делаем по одному токену за раз, каждый раз запуская полный forward pass.

**Задание 22.4** — Генерация текста

Реализуй генерацию с параметром top-k (выбираем только из k лучших токенов):

```python
def generate(model, tokenizer, prompt, max_new=50, temperature=0.8, top_k=40):
    context = tokenizer.encode(prompt)

    for _ in range(max_new):
        with torch.no_grad():
            logits = model(torch.tensor([context]))

        next_logits = logits[0, -1, :] / temperature

        # Top-k фильтрация: убираем всё кроме топ-k
        top_k_values, _ = torch.topk(next_logits, top_k)
        next_logits[next_logits < top_k_values[-1]] = float('-inf')

        probs = torch.softmax(next_logits, dim=0)
        next_id = torch.multinomial(probs, 1).item()
        context.append(next_id)

    return tokenizer.decode(context)
```

### Итоги главы

Полный путь токена — девять шагов:

```
1. "Кот сидел на"
   ↓ BPE токенизация
2. [7234, 15893, 422]
   ↓ Bipolar Codebook (STE)
3. (1, 3, 4096) биполярные векторы
   ↓ Cyclic Position Permutation (0 параметров)
4. (1, 3, 4096) с позиционной информацией
   ↓ ThoughtLoop: 3 × 8 блоков
   ↓   Мысль 1: Memory + Binding Attention + Controller
   ↓   Мысль 2: то же + gate_2 ≈ 0.53
   ↓   Мысль 3: то же + gate_3 ≈ 0.49
5. (1, 3, 4096) финальные представления
   ↓ @ codebook.T × output_scale
6. (1, 3, 32768) логиты
   ↓ softmax(logits[:, -1, :])
7. (32768,) вероятности
   ↓ семплирование
8. next_token_id = 8831
   ↓ decode
9. "коврике"
```

Форма `(B, T, 4096)` сохраняется на протяжении почти всего пути. Вся магия — внутри этого постоянного пространства.
