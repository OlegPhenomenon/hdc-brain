# Глава 10. Позиционное кодирование — порядок слов имеет значение

---

Представь что тебе прислали конверт с тремя карточками внутри:

```
СОБАКА  УКУСИЛА  ЧЕЛОВЕКА
```

Три карточки. Лежат в конверте вперемешку. Ты вытряхиваешь их — и смотришь на три слова.

Но в каком порядке они были? "Собака укусила человека" — одно дело.
"Человека укусила собака" — смысл противоположный. И это не шутка: в суде важно кто кого укусил.

Вот именно такая проблема стоит перед нейросетью.

---

### Почему нейросеть не знает порядок

В предыдущей главе мы узнали: токенайзер превращает текст в числа.
В главе 11 узнаем: кодбук превращает числа в векторы.

Получается матрица:

```
"Собака укусила человека"

Строка 0: [вектор для "Собака"]
Строка 1: [вектор для "укусила"]
Строка 2: [вектор для "человека"]
```

Нейросеть обрабатывает эту матрицу. Но матрица — это просто набор строк.
Строки **не упорядочены** сами по себе.

Если перетасовать строки местами — нейросеть получит ту же информацию, но в другом порядке.
Без специальной пометки она не поймёт разницы между "собака укусила" и "укусила собака".

Нужно **пометить каждый вектор его позицией** в предложении.

---

### Первый способ: выучить позицию как часть словаря

Самое простое решение: добавим к каждому токену ещё один вектор — вектор позиции.

```python
# Таблица позиций: позиция → вектор
position_embedding = nn.Embedding(512, 4096)
# 512 позиций × 4096 размерность = 2 млн параметров

# Применение:
token_emb = codebook[idx]            # (B, T, D) — векторы токенов
positions = torch.arange(T)          # [0, 1, 2, ..., T-1]
pos_emb = position_embedding(positions)  # (T, D) — векторы позиций
x = token_emb + pos_emb              # складываем!
```

Токен "собака" на позиции 0 получит вектор = `[вектор_собаки] + [вектор_позиции_0]`.
На позиции 3 — другой вектор: `[вектор_собаки] + [вектор_позиции_3]`.

Нейросеть видит разницу. Задача решена.

Так работал оригинальный GPT (2018). Просто и хорошо.

**Одна проблема:** если текст длиннее чем 512 токенов — позиция 600 не была обучена.
Это как спросить повара о блюде 701, если в меню только 700.

### Второй способ: математическая формула (Sinusoidal)

В оригинальной статье про Transformer (2017) придумали изящное решение:
**не учить** векторы позиций, а **вычислять** их по формуле.

Идея: закодировать позицию как набор синусов и косинусов разных частот.

Аналогия: часы. У часов три стрелки — секундная, минутная, часовая.
Каждая показывает одно и то же время, но с разной "частотой" вращения.
По трём стрелкам вместе можно однозначно определить время.

```
Позиция  0: sin(0)=0.000, cos(0)=1.000, sin(0/100)=0.000, cos(0/100)=1.000 ...
Позиция  1: sin(1)=0.841, cos(1)=0.540, sin(1/100)=0.010, cos(1/100)=0.999 ...
Позиция  5: sin(5)=-0.96, cos(5)=0.284, sin(5/100)=0.050, cos(5/100)=0.998 ...
```

Каждая позиция = уникальный паттерн из синусов/косинусов.
Формула работает для **любого** числа, не только для тех что видели при обучении.

```python
import math
import torch

def sinusoidal_encoding(max_len, d_model):
    """Вычисляем sinusoidal позиционные кодировки."""
    pe = torch.zeros(max_len, d_model)
    position = torch.arange(0, max_len).float().unsqueeze(1)  # (T, 1)

    # "Делитель" определяет частоту для каждого измерения
    div_term = torch.exp(
        torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
    )

    pe[:, 0::2] = torch.sin(position * div_term)  # чётные измерения → sin
    pe[:, 1::2] = torch.cos(position * div_term)  # нечётные → cos

    return pe

# Пример
pe = sinusoidal_encoding(10, 8)  # 10 позиций, 8 измерений
for pos in range(5):
    vals = [f"{v:.2f}" for v in pe[pos].tolist()]
    print(f"Позиция {pos}: [{', '.join(vals)}]")
```

Вывод:
```
Позиция 0: [0.00, 1.00, 0.00, 1.00, 0.00, 1.00, 0.00, 1.00]
Позиция 1: [0.84, 0.54, 0.10, 0.99, 0.01, 1.00, 0.00, 1.00]
Позиция 2: [0.91, -0.42, 0.20, 0.98, 0.02, 1.00, 0.00, 1.00]
Позиция 3: [0.14, -0.99, 0.30, 0.96, 0.03, 1.00, 0.00, 1.00]
```

Каждая строка уникальна. 0 параметров. Работает на любой длине.

### Третий способ: вращение (RoPE)

Современный стандарт. Используется в LLaMA, Mistral, SmolLM2.

**Идея:** вместо того чтобы *добавлять* позиционный вектор — *вращать* токен в пространстве.

Аналогия: представь вектор как стрелку компаса. Позиция — это угол поворота этой стрелки.
Токен "собака" на позиции 0 — стрелка указывает на север. На позиции 3 — повернули на 30 градусов.

```
Обычный подход:   x_новый = x + pos_emb        ← прибавляем
RoPE:             x_новый = rotate(x, θ × pos)  ← вращаем
```

Почему вращение лучше прибавления?

Когда нейросеть вычисляет "насколько слово A связано со словом B" (это называется attention),
RoPE гарантирует что результат зависит **только от разницы позиций** (A-B), а не от их абсолютных значений.

Это значит: если модель научилась "смотреть на 3 позиции назад" — это работает
и в начале предложения, и в конце, и в любом месте.

Мы не будем копать глубоко в математику RoPE — это отдельная тема.
Важно знать: RoPE существует, HDC-Brain его **не использует** (мы придумали своё).

### Наш способ: циклический сдвиг (0 параметров!)

А теперь — самое интересное. То что мы сделали в HDC-Brain.

Помнишь операцию **Permute** из главы 6? Это циклический сдвиг вектора.

```
Вектор [a, b, c, d, e]

Permute на 1:  [e, a, b, c, d]  ← последний становится первым
Permute на 2:  [d, e, a, b, c]
Permute на 3:  [c, d, e, a, b]
```

Идея проста: **позиция токена = количество сдвигов**.

```
Токен на позиции 0: сдвигаем на 0 → [a, b, c, d, e]  (не меняется)
Токен на позиции 1: сдвигаем на 1 → [e, a, b, c, d]
Токен на позиции 2: сдвигаем на 2 → [d, e, a, b, c]
Токен на позиции 3: сдвигаем на 3 → [c, d, e, a, b]
```

Один и тот же токен "собака" на разных позициях — разные векторы.
Нейросеть видит разницу. Задача решена.

**И при этом: ноль дополнительных параметров!**
Никакой таблицы, никаких выучиваемых весов. Просто математическая операция.

---

### Код циклического сдвига

```python
def _cyclic_position(self, x):
    """Добавляем позиционную информацию через циклический сдвиг."""
    B, T, D = x.shape
    # B = размер батча
    # T = длина последовательности (512 токенов)
    # D = размерность вектора (4096)

    # Позиции токенов: [0, 1, 2, ..., T-1]
    positions = torch.arange(T, device=x.device)

    # Для каждой позиции p — считаем новые индексы
    # Элемент на месте d берётся из позиции (d - p) % D
    indices = (
        torch.arange(D, device=x.device).unsqueeze(0)  # (1, D)
        - positions.unsqueeze(1)                        # (T, 1)
    ) % D
    # indices: (T, D) — для каждого токена, какой элемент куда идёт

    # Применяем к батчу
    return x.gather(2, indices.unsqueeze(0).expand(B, -1, -1))
```

Разберём `indices` на маленьком примере (D=5, T=3):

```
positions = [0, 1, 2]
dims      = [0, 1, 2, 3, 4]

Для pos=0: indices = (dims - 0) % 5 = [0, 1, 2, 3, 4]  → берём элементы 0,1,2,3,4 → не меняется
Для pos=1: indices = (dims - 1) % 5 = [4, 0, 1, 2, 3]  → берём элементы 4,0,1,2,3 → сдвиг на 1
Для pos=2: indices = (dims - 2) % 5 = [3, 4, 0, 1, 2]  → берём элементы 3,4,0,1,2 → сдвиг на 2
```

Проверяем на числах:

```python
import torch

x = torch.tensor([[[10, 20, 30, 40, 50]]])  # B=1, T=1, D=5
# Сдвигаем на позицию 2

B, T, D = 1, 1, 5
positions = torch.tensor([2])
indices = (torch.arange(D).unsqueeze(0) - positions.unsqueeze(1)) % D
# indices = [[3, 4, 0, 1, 2]]

result = x.gather(2, indices.unsqueeze(0).expand(B, -1, -1))
print(result)  # [[[40, 50, 10, 20, 30]]]
#                    ↑          ↑
#                  было 40 (позиция 3)  было 10 (позиция 0)
```

### Формула для любопытных

Если хочется записать математически:

```
Cyclic_Shift_p(v)[d] = v[(d - p) mod D]

где:
  v — исходный вектор (D-мерный)
  p — позиция токена (0, 1, 2, ...)
  d — индекс измерения (0, 1, ..., D-1)
  D — размерность вектора (4096 в нашей модели)
  mod — остаток от деления (чтобы "закольцовываться")
```

Вот что это буквально означает:
- `v[(d - p) mod D]` — берём элемент вектора с индексом `(d - p)`, обёрнутым по кругу
- Для `p = 0`: берём элемент `d` → ничего не меняется
- Для `p = 1`: элемент на месте `d` берётся из позиции `d-1` → сдвиг
- Для `p = D`: `(d - D) mod D = d` → вернулись в начало (цикл!)

Последнее важно: при длине последовательности > D (4096) векторы начнут "повторяться".
Позиция 4096 = позиция 0 = позиция 8192 = ...

Это ограничение нашего подхода. Но наш максимальный контекст = 512 токенов — намного меньше 4096.

### Сравниваем все подходы

| Подход | Доп. параметров | Макс. длина | Умеет "относительные позиции"? | Используется в |
|--------|:--------------:|:-----------:|:-------------------------------:|:---------------|
| Learned | max_len × D | Ограничена обучением | Нет | GPT-2 |
| Sinusoidal | **0** | Не ограничена | Нет | Transformer (2017) |
| RoPE | **0** | Не ограничена | **Да** | LLaMA, Mistral, SmolLM2 |
| Cyclic Permute | **0** | До D (4096) | Нет | **HDC-Brain v14.1** |

Наш подход — самый простой. 0 параметров, работает в рамках нашего контекста.

**Почему мы выбрали именно это?**

Cyclic Permutation — это буквально операция **Permute** из HDC-арифметики (глава 6).
В HDC Permute — стандартный способ кодировать порядок в последовательностях.
Мы просто использовали тот же инструмент, который уже есть в нашей архитектуре.
Никакого лишнего кода, никаких лишних концепций.

**Задание 10.1** — Наглядно

Напиши код который показывает: "собака" на позиции 0 и "собака" на позиции 3 — разные векторы после сдвига.

```python
import torch

# Вектор токена "собака" (просто для примера)
sobaka = torch.arange(8).float()  # [0, 1, 2, 3, 4, 5, 6, 7]

def cyclic_shift(v, p):
    D = len(v)
    indices = (torch.arange(D) - p) % D
    return v[indices]

pos0 = cyclic_shift(sobaka, 0)
pos3 = cyclic_shift(sobaka, 3)

print(f"Позиция 0: {pos0.tolist()}")
print(f"Позиция 3: {pos3.tolist()}")
print(f"Одинаковые? {torch.allclose(pos0, pos3)}")
```

### Итоги главы

Без позиционного кодирования нейросеть не знает порядок слов.

Три способа это исправить:
- **Learned**: учим вектор для каждой позиции → просто, ограничен
- **Sinusoidal**: математика (sin/cos) → гибкий, не адаптируется
- **RoPE**: вращение векторов → современный стандарт, относительные позиции

Наш способ:
- **Cyclic Permutation**: циклический сдвиг на p позиций → 0 параметров, HDC-нативный

```
Токен на позиции p:  x_shifted = roll(x, shift=-p)
```
