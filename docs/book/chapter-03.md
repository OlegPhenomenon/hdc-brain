## Антон и эквалайзер

Антон — звукорежиссёр. У него на пульте стоит эквалайзер — 10 ручек:

```
[низкие] ━━━━━━━━━━ [высокие]
  60Hz  120  250  500  1k  2k  4k  8k  16k  20k
```

Задача: настроить звук чтобы вокал "звучал правильно."

Антон крутит ручки. Слушает. Крутит снова. Слушает. Через час — идеальный звук.

Что он сделал? Подобрал **значения** (положения ручек) так чтобы **функция** (обработка звука) давала **нужный результат** (хороший вокал).

Нейронная сеть — это то же самое. Только ручек не 10 а **299 миллионов**. И вместо Антона — алгоритм который крутит ручки сам.

---

## Нейронная сеть = функция с параметрами

Самое простое определение:

```
Нейронная сеть = функция с настраиваемыми параметрами

f(вход, параметры) → выход

параметры = миллионы чисел (как ручки эквалайзера)
```

До обучения ручки стоят случайно — функция выдаёт мусор:

```
f("The capital of France is", случайные_ручки) → "banana"
```

После обучения ручки подобраны — функция работает:

```
f("The capital of France is", подобранные_ручки) → "Paris"
```

Что изменилось? Только **значения параметров**. Код тот же. Архитектура та же. Но числа внутри стали правильными.

В Rails это как `config/application.yml`: одна и та же программа, но с разными настройками работает по-разному. Только в нейросетях настроек — миллионы, и подбираются они автоматически.

---

## Linear слой: одна "ручка" нейросети

Запусти:

```python
import torch
import torch.nn as nn

# Linear: принимает вектор из 4 чисел, выдаёт вектор из 8
linear = nn.Linear(4, 8)

x = torch.tensor([[0.9, -0.7, 0.9, -0.8]])
y = linear(x)

print(f"Вход:  {x.shape}")         # (1, 4)
print(f"Выход: {y.shape}")         # (1, 8) — другой размер!
print(f"Выход: {y.data}")          # 8 чисел — какие-то случайные

# Заглянем внутрь:
print(f"\nМатрица весов: {linear.weight.shape}")    # (8, 4)
print(f"Смещение:      {linear.bias.shape}")        # (8,)
print(f"Параметров:    {4*8 + 8} = {4*8} весов + {8} смещений")
```

**Linear** — это слой который делает ровно одно: умножает вход на матрицу и прибавляет смещение.

```
y = x @ W + b
```

Ты уже знаешь каждую часть:
- `x @ W` — матричное умножение из главы 2 ("смотрим через линзу")
- `+ b` — **bias** ("байас", смещение) — просто вектор который прибавляется к результату

Bias — как `offset` в SQL-запросе: сдвигает результат на фиксированную величину. Без него функция всегда проходила бы через ноль.

Linear слой — это основной "кирпичик" нейросетей. Из него строится почти всё.

## Проблема: один Linear бесполезен

Посмотри:

```
x → Linear_1 → Linear_2 → Linear_3 → y
```

Математически: `y = x @ W1 @ W2 @ W3`

Но три матрицы перемноженные подряд = одна матрица: `W_combined = W1 @ W2 @ W3`.

Три слоя = один слой. Десять слоёв = один слой. Сколько ни ставь — всё равно одно умножение.

```python
# Докажем:
W1 = torch.randn(4, 8)
W2 = torch.randn(8, 6)
W3 = torch.randn(6, 4)

x = torch.randn(1, 4)

# Три слоя подряд
y_three = x @ W1 @ W2 @ W3

# Одна объединённая матрица
W_combined = W1 @ W2 @ W3
y_one = x @ W_combined

print(f"Разница: {(y_three - y_one).abs().max().item():.10f}")
# → 0.0000000000 (одинаковый результат!)
```

Нужно что-то что **нарушает линейность**. Чтобы три слоя не сводились к одному.

## Функция активации: нелинейная "дверь"

Между Linear слоями ставим нелинейную функцию. Представь охранника на входе в клуб:

Если число отрицательное — говорит "нет, ноль." Если положительное — пропускает как есть.

В нейросетях это называется **ReLU** — сокращение от **Rectified Linear Unit**, что можно перевести как "выпрямленная линейная единица". Звучит страшно, но суть простая: обрезаем отрицательные до нуля.

Формула: `ReLU(x) = max(0, x)`

Здесь `max(0, x)` — это и есть наш охранник: возвращает большее из нуля и числа.

Запусти и посмотри:

```python
import torch
import torch.nn.functional as F

x = torch.tensor([-3.0, -1.0, 0.0, 1.0, 3.0])

print("x:   ", x.tolist())
print("ReLU:", F.relu(x).tolist())
```

```
x:    [-3.0, -1.0, 0.0, 1.0, 3.0]
ReLU: [ 0.0,  0.0, 0.0, 1.0, 3.0]   ← отрицательные → ноль
```

```
Визуально:

выход
  │          ╱
  │         ╱
  │        ╱
  │       ╱
──┼──────●──────── вход
  │    0
  │
Слева от нуля: всё обрезано
Справа: y = x (прямая линия)
```

В HDC-Brain мы используем **GELU** — более мягкую версию. GELU расшифровывается как **Gaussian Error Linear Unit** — "линейная единица с гауссовой ошибкой." Отличие от ReLU: не резко обрезает, а плавно "затухает". Как охранник который иногда с неохотой пропускает даже тех кто чуть-чуть не дотягивает:

```python
print("GELU:", [f"{v:.2f}" for v in F.gelu(x).tolist()])
```

```
GELU: [-0.00, -0.16, 0.00, 0.84, 3.00]
         ↑      ↑                   ↑
    почти 0   не совсем 0     почти 3 (пропустил)
```

GELU работает лучше для языковых моделей: нейрон не "умирает" полностью при отрицательном входе.

---

## Теперь три слоя работают

С функцией активации между слоями — три слоя уже НЕ сводятся к одному:

```
x → Linear_1 → GELU → Linear_2 → GELU → Linear_3 → y
```

GELU "ломает" линейность. Каждый слой делает что-то уникальное. Чем больше слоёв — тем более сложные паттерны может найти сеть.

---

## Residual connection: хороший редактор

Посмотри на фрагмент реального кода HDC-Brain:

```python
return x + h   # ← ЭТА СТРОКА — ключевая!
```

`x` — входные данные. `h` — результат обработки. Мы **складываем** их вместо того чтобы заменять.

Зачем? Аналогия: два типа редакторов.

**Плохой редактор**: берёт твою статью и полностью переписывает. Если облажался — оригинал потерян навсегда.

**Хороший редактор**: берёт статью и делает пометки на полях. "Тут добавить абзац. Тут убрать повтор." Оригинал сохранён. Пометки — улучшения.

`return x + h` — это хороший редактор. `h` — пометки на полях.

Это называется **Residual Connection** — "остаточное соединение." Слово "residual" переводится как "остаточный" — потому что слой учится предсказывать не полный результат, а **остаток**: разницу между текущим состоянием и идеальным.

```
          ┌──────────────────────────────────┐
          │   (прямой канал — "ничего не делай")   │
          │                                  │
x ────────┼──→ обработка → h                 │
          │                                  │
          └──────────── + ──────────────────┘
                        │
                      x + h = выход
```

Без residual connections глубокие сети (30+ слоёв) почти не обучаются. С ними — работают отлично. Это одно из главных открытий в deep learning (2015 год, ResNet).

Запусти и посмотри:

```python
import torch, torch.nn as nn, torch.nn.functional as F

dim = 128
x = torch.randn(1, dim)

# Слой без residual
layer = nn.Linear(dim, dim)
y_without = layer(x)

# Слой с residual
y_with = x + layer(x)

print(f"Вход:            mean={x.mean():.3f}")
print(f"Без residual:    mean={y_without.mean():.3f}")
print(f"С residual:      mean={y_with.mean():.3f}")
# С residual ≈ сохраняет входную статистику + добавляет поправку
```

## ControllerBlock: собираем вместе

В HDC-Brain есть блок который использует всё что мы разобрали. Вот он целиком:

```python
class ControllerBlock(nn.Module):
    def __init__(self, hdc_dim, inner_dim, dropout=0.1):
        super().__init__()
        self.ln   = nn.LayerNorm(hdc_dim)       # нормализация
        self.down = nn.Linear(hdc_dim, inner_dim) # 4096 → 2560
        self.up   = nn.Linear(inner_dim, hdc_dim) # 2560 → 4096
        self.drop = nn.Dropout(dropout)

    def forward(self, x):
        h = self.ln(x)                # нормализуем числа
        h = F.gelu(self.down(h))      # сжимаем + GELU
        h = self.drop(self.up(h))     # расширяем обратно
        return x + h                  # residual!
```

Разберём построчно:

**`self.ln = nn.LayerNorm`** — нормализация. Приводит все числа к "нормальному" виду (среднее ≈ 0, разброс ≈ 1). Без этого числа могут "взорваться" после многих умножений: 0.1 → 8 → 730 → 45000 → NaN. LayerNorm — как предохранитель в электрической цепи. Подробно разберём в главе 15.

**`self.down = nn.Linear(4096, 2560)`** — "бутылочное горлышко". Сжимаем 4096 чисел в 2560. Это как pересказать книгу в трёх предложениях — выделяешь главное, отбрасываешь мусор.

**`F.gelu(...)`** — нелинейность. Без неё два Linear подряд = один Linear.

**`self.up = nn.Linear(2560, 4096)`** — расширяем обратно. Из "пересказа" восстанавливаем полный вектор, но уже "переосмысленный."

**`nn.Dropout(0.1)`** — во время обучения случайно "выключаем" 10% чисел (ставим в ноль). Это как тренировка футбольной команды когда каждую игру один случайный игрок не выходит — все учатся не зависеть от одного звёздного игрока. Предотвращает переобучение (когда модель запоминает данные вместо того чтобы понимать паттерны). Подробно — в главе 15.

**`return x + h`** — residual. Хороший редактор.

```
Вход:  (12, 512, 4096) ← 12 предложений, 512 токенов, 4096 измерений
                         ↓
Выход: (12, 512, 4096) ← ФОРМА НЕ ИЗМЕНИЛАСЬ

Это позволяет ставить блоки один за другим:
x → Block_1 → Block_2 → ... → Block_8 → выход
```

### Сколько параметров?

```
LayerNorm:         2 × 4096                  =     8 192
Linear(down): 4096 × 2560 + 2560            = 10 488 320
Linear(up):   2560 × 4096 + 4096            = 10 489 856
───────────────────────────────────────────────
ИТОГО на один блок:                        ≈ 21 000 000
8 блоков:                                  ≈ 168 000 000 параметров

Это 56% всей модели!
```

## Итоги

```
Нейронная сеть = функция с параметрами (ручками эквалайзера)
Linear         = умножение на матрицу + смещение (основной кирпичик)
GELU           = нелинейная дверь между слоями (мягкий охранник)
Residual       = x + block(x)  (хороший редактор: улучшай, не переписывай)
ControllerBlock = всё вместе: norm → сжать → GELU → расширить → +оригинал
```

8 таких блоков = 168 миллионов параметров = 56% всей HDC-Brain.
