# Глава 9. Токенизация — как разбить текст на кусочки

---

Саша работает переводчиком.

Однажды ему дали задание: перевести огромную книгу. 800 страниц.

Первая идея: переводить **побуквенно**. Буква за буквой. Это точно, но безумно медленно —
на каждую страницу уходит день.

Вторая идея: переводить **постранично**. Целая страница за раз. Быстро!
Но проблема — редкие технические термины. Целая страница = один "кусок".
Что делать, если страница содержит слово, которого нет в словаре?
Пишем `[НЕИЗВЕСТНО]` и теряем смысл.

Третья идея, которую придумал Саша: **переводить по смысловым кускам**.
Обычные слова — целиком. Редкие — по частям (корень + суффикс + окончание).
Нет такого понятия "неизвестное слово" — любое слово раскладывается на знакомые куски.

---

### Та же проблема у нейросети

Нейросеть работает с числами.
Текст — это символы: буквы, пробелы, знаки препинания.
Нужно превратить текст в числа.

**Вариант 1: побуквенно**

```
"Hello" → [H, e, l, l, o] → [72, 101, 108, 108, 111]
```

Просто, работает. Но предложение из 100 символов = 100 токенов.
Нейросеть должна "запомнить" связь между первым и последним символом слова —
это сложно.

**Вариант 2: по словам**

```
"Hello world" → ["Hello", "world"] → [4521, 8203]
```

Лучше. Но что если встречается слово "нейрохирургический"? Его нет в словаре.
Пишем `[UNK]` — "неизвестный токен". Теряем информацию.

**Правильный ответ: между символами и словами**

Именно это и делает **токенизация**. А конкретный алгоритм, который все используют — **BPE**.

---

### История BPE

В 1994 году Philip Gage придумал **Byte-Pair Encoding (BPE)** — алгоритм **сжатия данных**.
Идея: если в тексте часто встречается пара байт `"th"` — замени её одним новым символом.
Повторяй пока не уменьшишь файл достаточно.

В 2016 году Rico Sennrich взял этот алгоритм сжатия и применил к задаче токенизации в NLP.
Идея: используй BPE не для сжатия, а для построения словаря "подслов".

Получилось отлично. Сейчас BPE используют GPT-4, Claude, LLaMA — все крупные модели.

---

### Как работает BPE: пошагово

Начнём с небольшого примера.

Допустим, у нас есть текст:
```
"low low low lower lower newest newest newest widest widest"
```

**Шаг 0: Считаем частоты слов**

```
"low"    × 3
"lower"  × 2
"newest" × 3
"widest" × 2
```

**Разбиваем каждое слово на буквы**, добавляем `</w>` (маркер конца слова):

```
l o w </w>        × 3
l o w e r </w>    × 2
n e w e s t </w>  × 3
w i d e s t </w>  × 2
```

**Шаг 1: Ищем самую частую пару**

Считаем все встречающиеся пары:

```
(l, o)    : 3 + 2 = 5 раз
(o, w)    : 3 + 2 = 5 раз
(e, s)    : 3 + 2 = 5 раз
(w, </w>) : 3 раза
...
```

Берём `(e, s)` — частота 5. Объединяем `e` + `s` → `es`:

```
l o w </w>         × 3
l o w e r </w>     × 2
n e w es t </w>    × 3   ← объединили!
w i d es t </w>    × 2   ← и здесь!
```

**Шаг 2: Снова ищем самую частую пару**

```
(es, t): 3 + 2 = 5 раз  ← самая частая
```

Объединяем `es` + `t` → `est`:

```
l o w </w>          × 3
l o w e r </w>      × 2
n e w est </w>      × 3
w i d est </w>      × 2
```

**Шаг 3, 4, 5...** и так далее.

После нескольких итераций:
```
low</w>              × 3
l o w e r </w>       × 2
newest</w>           × 3
w i d est </w>       × 2
```

Слово `"low"` собралось в один токен. Слово `"newest"` тоже.
Слово `"widest"` — пока нет, будет собрано позже.

В реальности алгоритм делает **32,000 итераций** и обучается на миллиардах слов.
После этого большинство обычных слов = 1 токен.
Редкие слова = 2-3 токена (раскладываются на части).

### Код BPE с нуля

```python
from collections import Counter

def get_vocab(corpus):
    """Разбиваем корпус на слова, слова на символы."""
    vocab = Counter()
    for word in corpus.split():
        # Каждое слово: символы через пробел + маркер конца </w>
        chars = ' '.join(list(word)) + ' </w>'
        vocab[chars] += 1
    return vocab

def get_pairs(vocab):
    """Находим все пары рядом стоящих символов и их частоты."""
    pairs = Counter()
    for word, freq in vocab.items():
        symbols = word.split()
        for i in range(len(symbols) - 1):
            pairs[(symbols[i], symbols[i+1])] += freq
    return pairs

def merge_vocab(pair, vocab):
    """Объединяем заданную пару в один токен во всём словаре."""
    new_vocab = {}
    bigram = ' '.join(pair)        # "e s"
    replacement = ''.join(pair)    # "es"
    for word, freq in vocab.items():
        new_word = word.replace(bigram, replacement)
        new_vocab[new_word] = freq
    return new_vocab

def train_bpe(corpus, num_merges=10):
    """Обучаем BPE токенайзер."""
    vocab = get_vocab(corpus)
    merges = []

    print("Начальный словарь:")
    for word, freq in vocab.items():
        print(f"  '{word}': {freq}")
    print()

    for i in range(num_merges):
        pairs = get_pairs(vocab)
        if not pairs:
            break

        # Самая частая пара
        best = max(pairs, key=pairs.get)
        merges.append(best)

        vocab = merge_vocab(best, vocab)
        print(f"Слияние {i+1}: '{best[0]}' + '{best[1]}' → '{''.join(best)}' (частота {pairs[best]})")

    return vocab, merges

# Запускаем
corpus = "low low low lower lower newest newest newest widest widest"
vocab, merges = train_bpe(corpus, num_merges=8)

print("\nФинальный словарь:")
for word, freq in vocab.items():
    print(f"  '{word}': {freq}")
```

Вывод:
```
Слияние 1: 'e' + 's' → 'es' (частота 5)
Слияние 2: 'es' + 't' → 'est' (частота 5)
Слияние 3: 'l' + 'o' → 'lo' (частота 5)
Слияние 4: 'lo' + 'w' → 'low' (частота 5)
Слияние 5: 'n' + 'e' → 'ne' (частота 3)
Слияние 6: 'ne' + 'w' → 'new' (частота 3)
Слияние 7: 'new' + 'est' → 'newest' (частота 3)
Слияние 8: 'low' + '</w>' → 'low</w>' (частота 3)
```

### Почему 32,000 токенов?

Это баланс между двумя крайностями:

```
Мало токенов (например, 1000):
  + Меньше параметров в кодбуке
  - "Нейросеть" превращается в ["ней", "ро", "сеть"] — 3 токена
  - Текст становится длиннее → сложнее обучать

Много токенов (например, 200,000):
  + Каждое слово = один токен
  - Кодбук огромный (200K × 4096 = 800M параметров!)
  - Редкие токены почти не встречаются → плохо обучены

32,000 токенов — золотая середина:
  Кодбук = 32K × 4096 = 131M параметров (приемлемо)
  Обычные слова = 1 токен
  Редкие слова = 2-3 токена
```

Для сравнения:
```
GPT-2:      50K токенов
GPT-4:     100K токенов
LLaMA:      32K токенов
HDC-Brain v14.1:  32K токенов
```

### SentencePiece: библиотека для реального использования

В нашем коде используется библиотека **SentencePiece** от Google.
Это BPE со множеством улучшений для production.

Обучение токенайзера:

```python
import sentencepiece as spm

spm.SentencePieceTrainer.train(
    input='train_text.txt',    # наш текстовый корпус
    model_prefix='bpe_en_32k', # имя файла модели
    vocab_size=32000,          # размер словаря
    model_type='bpe',          # алгоритм BPE
    character_coverage=0.9999, # покрываем 99.99% символов
    pad_id=0,                  # <pad> = id 0
    bos_id=1,                  # <bos> = id 1 (начало текста)
    eos_id=2,                  # <eos> = id 2 (конец текста)
    unk_id=3,                  # <unk> = id 3 (неизвестный символ)
)
```

Использование:

```python
sp = spm.SentencePieceProcessor(model_file='bpe_en_32k.model')

# Текст → числа
ids = sp.encode("The capital of France is Paris")
print(ids)    # [450, 5473, 310, 6355, 338, 3681]

# Числа → текст
text = sp.decode(ids)
print(text)   # "The capital of France is Paris"

# Посмотреть на кусочки
pieces = sp.encode_as_pieces("The capital of France is Paris")
print(pieces) # ['▁The', '▁capital', '▁of', '▁France', '▁is', '▁Paris']
```

Знак `▁` (маленький квадратик внизу) означает "здесь был пробел".
SentencePiece кодирует пробелы прямо внутри токенов — очень удобно.

---

### Что происходит с разными словами

```python
# Обычные слова — один токен
print(sp.encode_as_pieces("The quick fox"))
# ['▁The', '▁quick', '▁fox']

# Длинное редкое слово — несколько токенов
print(sp.encode_as_pieces("supercalifragilistic"))
# ['▁super', 'cali', 'frag', 'il', 'istic']

# Числа — тоже несколько токенов
print(sp.encode_as_pieces("12345"))
# ['▁123', '45']   или  ['▁1', '2', '3', '4', '5']

# Код — разбивается интересно
print(sp.encode_as_pieces("torch.nn.Module"))
# ['▁torch', '.', 'nn', '.', 'Module']
```

Один токен = в среднем **4-5 символов** для английского текста.

### Специальные токены

В кодбуке первые 4 позиции зарезервированы:

```
id = 0  →  <pad>   — "заполнитель" (когда тексты разной длины, короткие дополняем)
id = 1  →  <bos>   — Beginning of Sequence (начало текста)
id = 2  →  <eos>   — End of Sequence (конец текста)
id = 3  →  <unk>   — Unknown (символ, которого нет в словаре — почти не используется)
```

Пример использования в нашей модели:

```python
# Кодируем предложение
text = "Hello world"
ids = sp.encode(text)  # [450, 8203]

# Добавляем служебные токены
ids_with_special = [BOS_ID] + ids + [EOS_ID]
# [1, 450, 8203, 2]

# Модель обучается предсказывать следующий токен:
# Input:  [1, 450, 8203]      ← <bos> Hello world
# Target: [450, 8203, 2]      ←  Hello world <eos>
```

### Как это используется в HDC-Brain

В нашем коде `prepare.py`:

```python
# Шаг 1: Обучаем токенайзер (один раз)
spm.SentencePieceTrainer.train(
    input='data/train.txt',
    model_prefix='bpe_en_32k',
    vocab_size=32000,
    ...
)

# Шаг 2: Токенизируем весь корпус
sp = spm.SentencePieceProcessor(model_file='bpe_en_32k.model')

with open('data/train.txt') as f:
    text = f.read()

ids = sp.encode(text)
# ids = [450, 8203, 1234, 5678, ...]  — список из миллиардов чисел

# Шаг 3: Сохраняем в бинарный файл (быстро читать при обучении)
import numpy as np
arr = np.array(ids, dtype=np.uint16)  # uint16 = 2 байта, хватает для 32K
arr.tofile('data/train.bin')
```

При обучении:

```python
# Читаем кусок из файла
data = np.memmap('data/train.bin', dtype=np.uint16, mode='r')

# Берём случайный кусок длиной 512 токенов
i = random.randint(0, len(data) - 512 - 1)

# x[0] → предсказываем x[1]
# x[1] → предсказываем x[2]
# ...
# x[511] → предсказываем x[512]
```

### Формула токенизации (на самом деле не формула)

Токенизация — это **алгоритм**, не формула. Но можно записать так:

```
Токенайзер T: текст → последовательность чисел

T("Hello world") = [id_1, id_2, ..., id_n]

где id_i ∈ {0, 1, ..., V-1}   (V = размер словаря = 32,000)
```

Вот что это означает буквально:
- `T` — функция, которую мы **обучили** на корпусе
- Один и тот же текст всегда даёт одинаковые id
- Любой текст можно закодировать (нет "неизвестных" слов — раскладываем на части)
- По id всегда можно восстановить исходный текст обратно

### Итоги главы

Токенизация — мост между текстом и числами:

```
Текст:   "Hello world"
         ↓ токенайзер (BPE)
Числа:   [450, 8203]
         ↓ кодбук
Векторы: [[0.3, -0.7, ...], [0.1, 0.5, ...]]
         ↓ модель
Предсказание следующего токена
```

- **BPE**: начинаем с символов, многократно объединяем самую частую пару
- **32K токенов**: золотая середина размера словаря
- **SentencePiece**: production BPE, используется в HDC-Brain
- Один токен ≈ 4-5 символов для английского
- Специальные токены: `<pad>`, `<bos>`, `<eos>`, `<unk>`
